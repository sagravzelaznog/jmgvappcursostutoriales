Proyecto Final: Diseño e Implementación de una Red Empresarial

Objetivos del Proyecto

  • Aplicar todos los conceptos aprendidos en el curso
  • Diseñar una red empresarial completa
  • Configurar dispositivos de red según los requisitos
  • Implementar medidas de seguridad
  • Documentar el diseño y la configuración

Escenario del Proyecto

Una empresa de desarrollo de software con 3 departamentos (Desarrollo, Ventas y Administración) necesita implementar una red segura y eficiente. La empresa tiene una oficina principal y una sucursal remota que deben estar conectadas.

Requisitos de la Red

  • Oficina Principal:
    • Departamento de Desarrollo: 20 empleados
    • Departamento de Ventas: 10 empleados
    • Departamento de Administración: 5 empleados
    • Servidores: Web, Correo, Archivos
    • Acceso inalámbrico para visitantes
  • Sucursal Remota:
    • 10 empleados (todos en la misma red)
    • Conexión segura con la oficina principal
  • Requisitos de Conectividad:
    • Internet de alta velocidad
    • Red privada virtual (VPN) entre oficinas
    • Acceso remoto seguro para empleados

Instrucciones del Proyecto

1. Diseño de la Red

Diagrama de red sugerido:

[Inserte aquí un diagrama de red que muestre la topología propuesta]

Considere una topología jerárquica con:
  • Capa de Núcleo: Router principal conectado a Internet
  • Capa de Distribución: Switches L3 para el enrutamiento entre VLANs
  • Capa de Acceso: Switches L2 para conexión de dispositivos finales
  • Zona desmilitarizada (DMZ) para servidores accesibles desde Internet

2. Esquema de Direccionamiento IP

Diseñe un esquema de direccionamiento IP que incluya:

  • Subredes para cada departamento
  • Subred para servidores
  • Subred para dispositivos de red
  • Subred para la conexión entre routers
  • Espacio para crecimiento futuro
Considere usar el espacio de direcciones privadas 172.16.0.0/16 y dividirlo según las necesidades:
  • 172.16.1.0/24 - Red de administración
  • 172.16.10.0/24 - Departamento de Desarrollo
  • 172.16.20.0/24 - Departamento de Ventas
  • 172.16.30.0/24 - Departamento de Administración
  • 172.16.100.0/24 - Servidores
  • 172.16.200.0/30 - Enlace entre routers

3. Configuración de Dispositivos

Implemente la configuración necesaria en todos los dispositivos:

  • Configuración básica de routers y switches
  • Configuración de VLANs
  • Enrutamiento entre VLANs
  • Configuración de seguridad
  • Configuración inalámbrica

Configuración básica de VLAN en switches:

! Configuración de VLANs vlan 10 name Desarrollo exit vlan 20 name Ventas exit vlan 30 name Administracion exit vlan 100 name Servidores exit ! Configuración de puertos de acceso interface range fastEthernet 0/1-20 switchport mode access switchport access vlan 10 no shutdown exit interface range fastEthernet 0/21-30 switchport mode access switchport access vlan 20 no shutdown exit ! Configuración de puerto troncal al router interface gigabitEthernet 0/1 switchport mode trunk switchport trunk allowed vlan 10,20,30,100 no shutdown exit

4. Seguridad

Implemente las siguientes medidas de seguridad:

  • ACLs para controlar el tráfico entre redes
  • Seguridad de puertos en switches
  • SSH para administración remota
  • Autenticación AAA
  • Seguridad inalámbrica WPA2-Enterprise

Ejemplo de configuración de ACL extendida:

! ACL para permitir solo tráfico web y DNS desde la red de ventas a Internet access-list 100 permit tcp 172.16.20.0 0.0.0.255 any eq www access-list 100 permit tcp 172.16.20.0 0.0.0.255 any eq 443 access-list 100 permit udp 172.16.20.0 0.0.0.255 any eq domain access-list 100 deny ip 172.16.20.0 0.0.0.255 any ! Aplicar ACL a la interfaz de salida a Internet interface gigabitEthernet 0/1 ip access-group 100 out exit

5. Conectividad entre Oficinas

Configure una conexión VPN segura entre la oficina principal y la sucursal remota.

Configuración básica de VPN Site-to-Site con IPsec:

! Configuración en el router de la oficina principal crypto isakmp policy 10 encr aes 256 authentication pre-share group 5 crypto isakmp key cisco123 address 203.0.113.2 crypto ipsec transform-set MYSET esp-aes 256 esp-sha-hmac crypto map MYMAP 10 ipsec-isakmp set peer 203.0.113.2 set transform-set MYSET match address 100 ! Aplicar el mapa criptográfico a la interfaz de Internet interface gigabitEthernet 0/1 crypto map MYMAP exit ! Configurar lista de acceso para el tráfico a cifrar access-list 100 permit ip 172.16.0.0 0.0.255.255 192.168.1.0 0.0.0.255

Entregables

Para completar el proyecto, debe entregar:

  1. Documento de diseño de red que incluya:
    • Diagrama de topología de red
    • Esquema de direccionamiento IP
    • Descripción de las configuraciones implementadas
    • Medidas de seguridad implementadas
  2. Archivo de Packet Tracer (.pkt) con la implementación completa
  3. Documento de configuración de los dispositivos
  4. Manual de procedimientos para el administrador de red

Criterios de Evaluación

El proyecto será evaluado según los siguientes criterios:

Criterio Ponderación Descripción
Diseño de Red 20% Topología lógica y física adecuada
Configuración de Dispositivos 30% Configuración correcta de routers, switches y puntos de acceso
Seguridad 25% Implementación de medidas de seguridad adecuadas
Conectividad 15% Funcionamiento correcto de la conectividad entre dispositivos
Documentación 10% Completitud y claridad de la documentación

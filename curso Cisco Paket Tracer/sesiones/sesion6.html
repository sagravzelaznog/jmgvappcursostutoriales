Inicio Sesión 6

Protocolos de Enrutamiento Dinámico

Objetivos de Aprendizaje

  • Comprender los conceptos básicos de los protocolos de enrutamiento dinámico
  • Configurar y verificar RIP, OSPF y EIGRP
  • Entender las diferencias entre protocolos de vector distancia y estado de enlace
  • Solucionar problemas comunes en protocolos de enrutamiento dinámico

Introducción al Enrutamiento Dinámico

Los protocolos de enrutamiento dinámico permiten a los routers compartir automáticamente información sobre redes remotas. Esto facilita la administración de redes grandes y permite una recuperación automática ante fallos.

Tipos de Protocolos de Enrutamiento

  1. Vector Distancia (Distance Vector): RIP, IGRP
  2. Estado de Enlace (Link State): OSPF, IS-IS
  3. Híbrido: EIGRP

RIP (Routing Information Protocol)

Características de RIP

  • Métrica basada en conteo de saltos (máximo 15 saltos)
  • Actualizaciones periódicas cada 30 segundos
  • Versiones: RIPv1 (sin soporte VLSM) y RIPv2 (con soporte VLSM)
  • Uso de UDP puerto 520

OSPF (Open Shortest Path First)

Conceptos Clave de OSPF

  • Divide la red en áreas para mejorar la escalabilidad
  • Área 0 (troncal) conecta todas las demás áreas
  • Usa el algoritmo SPF (Dijkstra) para calcular rutas
  • Métrica basada en el ancho de banda

EIGRP (Enhanced Interior Gateway Routing Protocol)

Características de EIGRP

  • Protocolo avanzado de Cisco (ahora parcialmente abierto)
  • Usa DUAL (Diffusing Update Algorithm) para calcular rutas
  • Métrica compuesta (ancho de banda, retardo, carga, confiabilidad)
  • Actualizaciones incrementales

Práctica: Configuración de Protocolos de Enrutamiento

Ejercicio 1: Configuración Básica de RIPv2

Configura RIPv2 en una topología de tres routers.

  1. Crea una topología con tres routers conectados en cadena.
  2. Configura las direcciones IP en todas las interfaces.
  3. Habilita RIPv2 en todos los routers.
  4. Verifica las tablas de enrutamiento.

Configuración en cada router:

enable configure terminal router rip version 2 network [dirección-red] no auto-summary end copy running-config startup-config

Para verificar:

show ip route show ip protocols debug ip rip

Ejercicio 2: Configuración de OSPF

Configura OSPF en la misma topología.

  1. Elimina la configuración de RIP.
  2. Configura OSPF con el ID de proceso 1.
  3. Asigna todas las redes al área 0.
  4. Verifica la adyacencia de vecinos.

Configuración en cada router:

enable configure terminal router ospf 1 network [dirección-red] [wildcard-mask] area 0 end copy running-config startup-config

Para verificar:

show ip ospf neighbor show ip ospf interface show ip ospf database

Ejercicio 3: Configuración de EIGRP

Configura EIGRP en la misma topología.

  1. Elimina la configuración de OSPF.
  2. Configura EIGRP con el número de sistema autónomo 100.
  3. Habilita EIGRP en todas las interfaces.
  4. Verifica los vecinos EIGRP.

Configuración en cada router:

enable configure terminal router eigrp 100 network [dirección-red] [wildcard-mask] no auto-summary eigrp router-id [ID-del-router] end copy running-config startup-config

Para verificar:

show ip eigrp neighbors show ip eigrp topology show ip route eigrp

Resolución de Problemas

Algunos comandos útiles para solucionar problemas en protocolos de enrutamiento dinámico:

show ip protocols show ip route debug ip rip debug ip ospf events debug eigrp packets ping [dirección-ip] traceroute [dirección-ip]

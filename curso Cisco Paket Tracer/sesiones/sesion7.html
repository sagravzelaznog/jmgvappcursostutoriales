VLANs: Redes de Área Local Virtuales

Objetivos de Aprendizaje

  • Comprender el concepto y beneficios de las VLANs
  • Aprender a configurar VLANs en switches Cisco
  • Configurar enlaces troncales (trunks) con 802.1Q
  • Entender el protocolo VTP (VLAN Trunking Protocol)
  • Solucionar problemas comunes de VLANs

¿Qué son las VLANs?

Una VLAN (Virtual Local Area Network) es una subred lógica que agrupa dispositivos de red que están configurados para comunicarse como si estuvieran conectados al mismo dominio de difusión, independientemente de su ubicación física.

Beneficios de las VLANs

  • Segmentación de red: Mejora el rendimiento al reducir el tamaño de los dominios de difusión.
  • Seguridad: Aísla el tráfico entre diferentes departamentos o grupos.
  • Flexibilidad: Permite agrupar usuarios por función en lugar de por ubicación física.
  • Administración simplificada: Facilita la administración de la red.

Tipos de Puertos en un Switch

Tipo de Puerto Descripción Uso Común
Acceso (Access) Pertenece a una sola VLAN Conexión de dispositivos finales (PCs, impresoras)
Troncal (Trunk) Transporta múltiples VLANs Conexión entre switches o entre switch y router
Enojo (Voice) Configuración especial para voz y datos Conexión de teléfonos IP con PC conectada al teléfono

Protocolo VTP (VLAN Trunking Protocol)

VTP es un protocolo de Cisco que permite la administración centralizada de VLANs en una red de switches.

  • Modos VTP: Servidor, Cliente y Transparente
  • Dominio VTP: Grupo de switches que comparten información de VLANs
  • Versiones: V1, V2, V3 (la más reciente)

Práctica: Configuración de VLANs

Ejercicio 1: Creación de VLANs Básicas

Configura dos VLANs en un switch y asigna puertos a cada una.

  1. Crea las VLANs 10 (Ventas) y 20 (Marketing).
  2. Asigna los puertos 1-5 a la VLAN 10 y los puertos 6-10 a la VLAN 20.
  3. Verifica la configuración.
enable configure terminal ! Crear VLANs vlan 10 name Ventas exit vlan 20 name Marketing exit ! Asignar puertos a VLAN 10 interface range fastEthernet 0/1-5 switchport mode access switchport access vlan 10 no shutdown exit ! Asignar puertos a VLAN 20 interface range fastEthernet 0/6-10 switchport mode access switchport access vlan 20 no shutdown exit ! Verificar configuración show vlan brief show vlan id 10 show vlan id 20

Ejercicio 2: Configuración de Troncales

Configura un enlace troncal entre dos switches para transportar múltiples VLANs.

  1. Conecta dos switches con un cable cruzado.
  2. Configura el puerto de conexión en ambos switches como troncal.
  3. Verifica la configuración del troncal.

En ambos switches:

enable configure terminal ! Seleccionar el puerto de conexión entre switches interface fastEthernet 0/24 ! Configurar como troncal switchport mode trunk ! Especificar las VLANs permitidas (opcional) switchport trunk allowed vlan 10,20 ! Establecer el modo de encapsulación (generalmente negociación automática) switchport trunk encapsulation dot1q no shutdown exit ! Verificar configuración del troncal show interfaces trunk show interfaces fastEthernet 0/24 switchport

Ejercicio 3: Configuración de VTP

Configura el protocolo VTP entre dos switches.

  1. Configura un switch como servidor VTP y otro como cliente.
  2. Establece el mismo dominio VTP y contraseña en ambos switches.
  3. Verifica la sincronización de VLANs.

En el switch servidor:

enable configure terminal vtp domain MIEMPRESA vtp password cisco123 vtp mode server vtp version 2 end ! Verificar configuración VTP show vtp status show vtp password

En el switch cliente:

enable configure terminal vtp domain MIEMPRESA vtp password cisco123 vtp mode client vtp version 2 end ! Verificar sincronización show vlan brief show vtp status

Solución de Problemas Comunes

! Ver estado de las VLANs show vlan brief ! Ver configuración de un puerto específico show interfaces [interfaz] switchport ! Ver estado de los troncales show interfaces trunk ! Ver información VTP show vtp status show vtp password ! Ver tráfico de difusión por VLAN debug sw-vlan vtp events debug sw-vlan vlan events

Modelo OSI y TCP/IP

Objetivos de Aprendizaje

  • Comprender la estructura y propósito del modelo OSI
  • Entender las capas del modelo TCP/IP
  • Identificar las diferencias entre los modelos OSI y TCP/IP
  • Practicar la encapsulación de datos en Packet Tracer

El Modelo OSI

El Modelo OSI (Open Systems Interconnection) es un marco de referencia para la comunicación entre sistemas informáticos. Está dividido en 7 capas, cada una con funciones específicas:

Capa Nombre Función Principal Ejemplo
7 Aplicación Interfaz entre la aplicación y la red HTTP, FTP, SMTP
6 Presentación Traducción, encriptación, compresión JPEG, MPEG, SSL
5 Sesión Establece, gestiona y termina conexiones NetBIOS, RPC
4 Transporte Transferencia confiable de datos TCP, UDP
3 Red Enrutamiento y direccionamiento lógico IP, ICMP, ARP
2 Enlace de Datos Acceso al medio y control de flujo Ethernet, PPP, Switch
1 Física Transmisión de bits a través del medio Cable, Hub, Repetidor

El Modelo TCP/IP

El modelo TCP/IP es el estándar actual de Internet y consta de 4 capas:

  1. Aplicación: Combina las capas de aplicación, presentación y sesión del modelo OSI.
  2. Transporte: Equivalente a la capa de transporte del modelo OSI.
  3. Internet: Equivalente a la capa de red del modelo OSI.
  4. Acceso a la Red: Combina las capas de enlace de datos y física del modelo OSI.

Práctica: Visualización de Paquetes en Packet Tracer

Ejercicio 1: Simulación de Comunicación

En este ejercicio, crearás una red simple y observarás cómo viajan los paquetes a través de las diferentes capas.

  1. Crea una red con dos computadoras conectadas a un switch.
  2. Asigna direcciones IP a ambas computadoras (ejemplo: 192.168.1.1 y 192.168.1.2).
  3. Haz clic en "Simulation" en la esquina inferior derecha de Packet Tracer.
  4. Selecciona un paquete ICMP (ping) en la lista de filtros.
  5. Envía un ping de una computadora a la otra.
  6. Observa cómo se encapsula el paquete a través de las diferentes capas.

Al enviar un ping, podrás observar cómo el paquete viaja a través de las diferentes capas:

  1. Capa de Aplicación: El comando ping inicia el proceso.
  2. Capa de Transporte: Se crea un segmento con el protocolo ICMP.
  3. Capa de Red: Se añade la dirección IP de origen y destino.
  4. Capa de Enlace de Datos: Se añaden las direcciones MAC.
  5. Capa Física: Los bits se transmiten por el cable.

Ejercicio 2: Análisis de Paquetes

Analiza los paquetes en tránsito para entender mejor el proceso de encapsulamiento.

  1. En el modo de simulación, haz clic en "Capture/Forward" para avanzar paso a paso.
  2. Selecciona un paquete en tránsito y haz clic en "Inspect".
  3. Observa la información en cada capa del modelo OSI.
  4. Compara la información de las capas en diferentes momentos de la transmisión.

Responde las siguientes preguntas basadas en tu observación:

  1. ¿Qué información contiene el encabezado de la capa de red?
  2. ¿Cómo cambia la información de la capa de enlace de datos en diferentes puntos de la red?
  3. ¿Qué diferencias observas entre el paquete de solicitud y el de respuesta?

Introducción a la Seguridad en Redes

La seguridad en redes es fundamental para proteger la infraestructura y los datos contra accesos no autorizados, ataques y amenazas potenciales.

Principios Básicos de Seguridad

  • Confidencialidad: Solo usuarios autorizados pueden acceder a la información.
  • Integridad: La información no puede ser modificada por usuarios no autorizados.
  • Disponibilidad: Los recursos deben estar disponibles cuando se necesiten.

Listas de Control de Acceso (ACLs)

Tipos de ACLs

  • ACL Estándar: Filtra solo por dirección IP de origen (1-99, 1300-1999)
  • ACL Extendida: Filtra por dirección IP de origen/destino, protocolo y puerto (100-199, 2000-2699)
  • ACL Nombrada: Versión más reciente que usa nombres en lugar de números

Seguridad de Puertos en Switches

Características de Port Security

  • Limita las direcciones MAC que pueden conectarse a un puerto
  • Puede aprender direcciones MAC automáticamente o configurarse manualmente
  • Acciones ante violaciones: shutdown, restrict, protect

SSH para Administración Segura

SSH (Secure Shell) es un protocolo que proporciona una conexión segura para la administración remota de dispositivos de red.

Ventajas sobre Telnet:

  • Cifrado de datos
  • Autenticación segura
  • Integridad de datos

Práctica: Configuración de Seguridad

Ejercicio 1: Configuración de ACL Estándar

Crea una ACL estándar que permita el acceso a la red 192.168.1.0/24 solo desde la subred 10.0.0.0/24.

enable configure terminal ! Crear ACL estándar access-list 1 permit 10.0.0.0 0.255.255.255 access-list 1 deny any ! Aplicar ACL a la interfaz interface gigabitethernet 0/0 ip access-group 1 out exit ! Verificar configuración show access-lists show ip interface gigabitethernet 0/0

Ejercicio 2: Configuración de Seguridad de Puertos

Configura la seguridad de puertos en un switch para permitir solo una dirección MAC por puerto.

enable configure terminal ! Seleccionar el puerto a proteger interface fastEthernet 0/1 ! Habilitar modo acceso switchport mode access ! Habilitar port security switchport port-security ! Establecer el número máximo de direcciones MAC (1 por defecto) switchport port-security maximum 1 ! Configurar la acción ante violaciones (shutdown, restrict, protect) switchport port-security violation shutdown ! Opcional: Configurar la dirección MAC estática ! switchport port-security mac-address XXXX.XXXX.XXXX ! Habilitar sticky learning (aprende automáticamente la primera MAC) switchport port-security mac-address sticky no shutdown exit ! Verificar configuración show port-security show port-security interface fastEthernet 0/1

Ejercicio 3: Configuración de SSH

Configura SSH para la administración remota segura de un router.

enable configure terminal ! Configurar nombre de host y dominio hostname RouterSSH ip domain-name miempresa.com ! Generar claves criptográficas crypto key generate rsa ! Tamaño de la clave (recomendado 1024 o mayor) 1024 ! Configurar autenticación local username admin secret miclave123 line vty 0 4 transport input ssh login local exit ! Configurar versión SSH ip ssh version 2 ! Configurar tiempo de espera de inactividad ip ssh time-out 60 ip ssh authentication-retries 2 ! Deshabilitar Telnet line vty 0 15 no transport input telnet exit ! Verificar configuración show ip ssh show ssh

Para conectarse desde un cliente:

ssh -l admin dirección-ip-del-router

Buenas Prácticas de Seguridad

  • Cambiar las contraseñas predeterminadas
  • Usar contraseñas seguras (mínimo 8 caracteres, mayúsculas, minúsculas, números y caracteres especiales)
  • Habilitar el registro de eventos (logging)
  • Deshabilitar servicios innecesarios (HTTP, CDP, etc.)
  • Mantener el software actualizado
  • Implementar contraseñas de modo privilegiado
  • Habilitar autenticación local o con servidor AAA

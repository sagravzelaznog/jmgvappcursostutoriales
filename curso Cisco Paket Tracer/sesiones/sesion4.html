Fundamentos de Direccionamiento IPv4

Una dirección IPv4 es un número de 32 bits que identifica de manera única un dispositivo en una red. Se representa en formato decimal punteado, por ejemplo: 192.168.1.1.

Estructura de una dirección IP

Una dirección IP consta de dos partes:

  • Porción de red: Identifica la red a la que pertenece el dispositivo.
  • Porción de host: Identifica el dispositivo específico en la red.

Ejemplo: En la dirección 192.168.1.10/24 (donde /24 es la máscara de red):

  • Porción de red: 192.168.1
  • Porción de host: .10
  • Máscara de red: 255.255.255.0

Clases de Direcciones IP

Clase Rango Máscara por defecto Uso
A 1.0.0.0 - 126.255.255.255 255.0.0.0 Redes muy grandes
B 128.0.0.0 - 191.255.255.255 255.255.0.0 Redes medianas
C 192.0.0.0 - 223.255.255.255 255.255.255.0 Redes pequeñas
D 224.0.0.0 - 239.255.255.255 N/A Multicast
E 240.0.0.0 - 255.255.255.255 N/A Experimental

Subredes

El subnetting es la práctica de dividir una red en subredes más pequeñas. Esto se hace tomando prestados bits de la porción de host para crear más redes.

Ventajas del Subnetting

  • Mejora el rendimiento de la red
  • Reduce la congestión
  • Mejora la seguridad
  • Facilita la administración

Ejercicio 1: Creación de Subredes

Divide la red 192.168.1.0/24 en 4 subredes iguales.

  1. Determina cuántos bits necesitas para crear 4 subredes (2n ≥ 4).
  2. Calcula la nueva máscara de red.
  3. Identifica el rango de direcciones para cada subred.

Para crear 4 subredes necesitamos 2 bits (2² = 4).

Máscara original: 255.255.255.0 (/24)

Nueva máscara: 255.255.255.192 (/26)

Las subredes serían:

  1. 192.168.1.0/26 (rango: 192.168.1.1 - 192.168.1.62)
  2. 192.168.1.64/26 (rango: 192.168.1.65 - 192.168.1.126)
  3. 192.168.1.128/26 (rango: 192.168.1.129 - 192.168.1.190)
  4. 192.168.1.192/26 (rango: 192.168.1.193 - 192.168.1.254)

Ejercicio 2: Configuración en Packet Tracer

Configura dos subredes diferentes en Packet Tracer.

  1. Crea una topología con dos switches y un router.
  2. Configura el router con dos interfaces en diferentes subredes.
  3. Asigna direcciones IP a los dispositivos en cada subred.
  4. Verifica la conectividad entre las subredes.

Configuración del router:

enable
configure terminal
interface GigabitEthernet0/0
ip address 192.168.1.1 255.255.255.192
no shutdown
exit
interface GigabitEthernet0/1
ip address 192.168.1.65 255.255.255.192
no shutdown
exit
ip route 0.0.0.0 0.0.0.0 [dirección del router de salida]
end
copy running-config startup-config

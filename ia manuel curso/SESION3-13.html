Métricas Avanzadas de Evaluación

Precisión, Recall y F1-Score

  • Precisión: VP / (VP + FP)
  • Recall: VP / (VP + FN)
  • F1-Score: 2 * (Precisión * Recall) / (Precisión + Recall)

Curvas ROC y AUC

  • ROC: TPR vs FPR
  • AUC: Área bajo la curva ROC
  • Interpretación de valores AUC

Otras Métricas

  • Log Loss
  • Matriz de Confusión Multiclase
  • Métricas específicas por clase

Matriz de Confusión Detallada

Métrica Fórmula Interpretación
Exactitud (VP + VN) / Total Porcentaje de predicciones correctas
Precisión VP / (VP + FP) De los predichos como positivos, ¿cuántos lo son realmente?
Sensibilidad (Recall) VP / (VP + FN) De los reales positivos, ¿cuántos detectamos?
Especificidad VN / (VN + FP) Capacidad de detectar correctamente los negativos

Técnicas de Validación Avanzadas

Validación Cruzada Anidada

Para una evaluación más robusta de hiperparámetros:

  • Bucle externo: Validación cruzada
  • Bucle interno: Optimización de parámetros

Time Series Split

Para datos temporales:

  • Respetar el orden temporal
  • Train en datos pasados, test en futuros

Stratified K-Fold

Para conjuntos de datos desbalanceados:

  • Mantiene proporciones de clase
  • Mejor estimación del rendimiento
from sklearn.model_selection import GridSearchCV, cross_val_score, KFold
from sklearn.ensemble import RandomForestClassifier
import numpy as np

# Configuración de la validación cruzada anidada
inner_cv = KFold(n_splits=5, shuffle=True, random_state=42)
outer_cv = KFold(n_splits=5, shuffle=True, random_state=42)

# Hiperparámetros a optimizar
param_grid = {
    'n_estimators': [50, 100, 200],
    'max_depth': [None, 10, 20, 30],
    'min_samples_split': [2, 5, 10]
}

# Búsqueda de hiperparámetros
clf = GridSearchCV(
    estimator=RandomForestClassifier(random_state=42),
    param_grid=param_grid,
    cv=inner_cv,
    scoring='accuracy',
    n_jobs=-1
)

# Validación cruzada externa
nested_scores = cross_val_score(clf, X, y, cv=outer_cv, scoring='accuracy')
print(f"Precisión media: {nested_scores.mean():.3f} ± {nested_scores.std():.3f}")

from sklearn.model_selection import TimeSeriesSplit
from sklearn.ensemble import RandomForestRegressor
from sklearn.metrics import mean_squared_error
import numpy as np

# Configuración de Time Series Split
tscv = TimeSeriesSplit(n_splits=5)
model = RandomForestRegressor(n_estimators=100, random_state=42)

scores = []
for train_index, test_index in tscv.split(X):
    X_train, X_test = X[train_index], X[test_index]
    y_train, y_test = y[train_index], y[test_index]
    
    model.fit(X_train, y_train)
    y_pred = model.predict(X_test)
    score = np.sqrt(mean_squared_error(y_test, y_pred))
    scores.append(score)

print(f"RMSE medio: {np.mean(scores):.3f} ± {np.std(scores):.3f}")

Casos de Estudio Prácticos

Caso 1: Desbalance de Clases

Cuando una clase es mucho más frecuente que otras:

  • Muestreo (oversampling/undersampling)
  • Pesos de clase
  • Métricas apropiadas (F1, AUC-ROC)

Caso 2: Datos Temporales

Consideraciones especiales para series de tiempo:

  • No mezclar datos de diferentes períodos
  • Validación temporal (walk-forward)
  • Métricas específicas (sMAPE, MASE)

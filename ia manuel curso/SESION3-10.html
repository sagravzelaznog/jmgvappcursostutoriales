Introducción a la Reducción de Dimensionalidad

La reducción de dimensionalidad es una técnica fundamental en el aprendizaje automático que permite simplificar conjuntos de datos complejos al reducir el número de variables o características, manteniendo la mayor cantidad de información posible. Esta técnica es especialmente útil cuando se trabaja con datos de alta dimensionalidad, donde el número de características es significativamente mayor que el número de observaciones.

Problema de la Maldición de la Dimensionalidad

A medida que aumenta el número de dimensiones en un conjunto de datos, el volumen del espacio de características crece exponencialmente, lo que puede llevar a:

  • Sobredimensionamiento del modelo
  • Mayor costo computacional
  • Mayor riesgo de sobreajuste
  • Dificultad para visualizar los datos

Beneficios de la Reducción de Dimensionalidad

  • Reducción del tiempo de entrenamiento
  • Mejora en la visualización de datos
  • Reducción del ruido en los datos
  • Mejora en el rendimiento del modelo
  • Prevención del sobreajuste

La reducción de dimensionalidad no solo es útil para la visualización, sino que también puede mejorar significativamente el rendimiento de los algoritmos de aprendizaje automático al eliminar características redundantes o irrelevantes.

Métodos de Reducción de Dimensionalidad

Existen dos enfoques principales para la reducción de dimensionalidad:

1. Extracción de Características

Transforma los datos en un espacio de menor dimensión mediante la combinación o transformación de las características existentes.

  • PCA (Análisis de Componentes Principales): Encuentra las direcciones de máxima varianza en los datos.
  • LDA (Análisis Discriminante Lineal): Maximiza la separabilidad entre clases.
  • t-SNE (t-Distributed Stochastic Neighbor Embedding): Ideal para visualización de datos de alta dimensión.
  • UMAP (Uniform Manifold Approximation and Projection): Alternativa más rápida a t-SNE que preserva mejor la estructura global.

2. Selección de Características

Selecciona un subconjunto de las características más relevantes del conjunto original.

  • Métodos de filtrado: Seleccionan características basadas en puntuaciones estadísticas.
  • Métodos de envoltura: Utilizan un modelo de aprendizaje automático para evaluar subconjuntos de características.
  • Métodos integrados: Realizan la selección de características durante el proceso de entrenamiento del modelo.

La elección del método de reducción de dimensionalidad depende del tipo de datos, el problema específico y los requisitos de interpretabilidad.

Análisis de Componentes Principales (PCA)

El PCA es uno de los métodos de reducción de dimensionalidad más utilizados. Funciona identificando los ejes (componentes principales) a lo largo de los cuales los datos tienen la mayor varianza.

Conceptos Clave del PCA

  • Componentes principales: Nuevas variables no correlacionadas que capturan la mayor variabilidad en los datos.
  • Varianza explicada: Porcentaje de la varianza total que explica cada componente principal.
  • Autovalores y autovectores: Los autovectores representan las direcciones de los componentes principales, y los autovalores representan su magnitud.

Implementación de PCA en Python

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA

# Cargar el conjunto de datos de ejemplo (Iris)
data = load_iris()
X = data.data
y = data.target
feature_names = data.feature_names

# Estandarizar los datos
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Aplicar PCA
pca = PCA(n_components=2)  # Reducir a 2 dimensiones
X_pca = pca.fit_transform(X_scaled)

# Crear un DataFrame con los componentes principales
pca_df = pd.DataFrame(data=X_pca, columns=['Componente Principal 1', 'Componente Principal 2'])
pca_df['Clase'] = y

# Visualizar los datos en el espacio de componentes principales
plt.figure(figsize=(10, 6))
for target, color in zip([0, 1, 2], ['r', 'g', 'b']):
    indices = pca_df['Clase'] == target
    plt.scatter(pca_df.loc[indices, 'Componente Principal 1'],
                pca_df.loc[indices, 'Componente Principal 2'],
                c=color, label=data.target_names[target])

plt.xlabel('Componente Principal 1 (Varianza: {:.2f}%)'.format(pca.explained_variance_ratio_[0] * 100))
plt.ylabel('Componente Principal 2 (Varianza: {:.2f}%)'.format(pca.explained_variance_ratio_[1] * 100))
plt.title('PCA del Conjunto de Datos Iris')
plt.legend()
plt.grid(True)
plt.show()

# Varianza explicada por cada componente
print("Varianza explicada por cada componente:", pca.explained_variance_ratio_)
print("Varianza acumulada:", np.cumsum(pca.explained_variance_ratio_))

# Visualizar la varianza explicada
plt.figure(figsize=(10, 4))
plt.bar(range(1, len(pca.explained_variance_ratio_) + 1), pca.explained_variance_ratio_, 
        alpha=0.5, align='center', label='Varianza individual explicada')
plt.step(range(1, len(pca.explained_variance_ratio_) + 1), np.cumsum(pca.explained_variance_ratio_),
         where='mid', label='Varianza acumulada explicada')
plt.ylabel('Proporción de varianza explicada')
plt.xlabel('Componentes principales')
plt.title('Varianza explicada por los componentes principales')
plt.legend(loc='best')
plt.tight_layout()
plt.show()

Explicación del Código

  1. Carga de datos: Cargamos el conjunto de datos de Iris, que contiene medidas de flores de diferentes especies.
  2. Estandarización: Estandarizamos los datos para que tengan media 0 y desviación estándar 1, lo que es importante para PCA.
  3. Aplicación de PCA: Creamos una instancia de PCA especificando que queremos reducir a 2 dimensiones y ajustamos el modelo a nuestros datos.
  4. Visualización: Visualizamos los datos transformados en el espacio de los dos primeros componentes principales.
  5. Análisis de varianza: Calculamos y visualizamos la varianza explicada por cada componente principal.

La varianza explicada nos ayuda a entender cuánta información retienen los componentes principales. En este caso, los dos primeros componentes principales capturan la mayor parte de la varianza en los datos.

Interpretación de los Componentes

Los componentes principales son combinaciones lineales de las características originales. Podemos examinar los pesos (loadings) de cada característica en los componentes principales:

# Obtener los componentes principales (autovectores)
components = pca.components_

# Crear un DataFrame para visualizar los pesos
loadings = pd.DataFrame(components.T, 
                        columns=['PC1', 'PC2'],
                        index=feature_names)

print("Pesos de las características en los componentes principales:")
print(loadings)

# Visualizar la importancia de las características en los componentes principales
plt.figure(figsize=(10, 6))
plt.scatter(loadings['PC1'], loadings['PC2'])

# Añadir etiquetas a los puntos
for i, feature in enumerate(feature_names):
    plt.annotate(feature, (loadings['PC1'][i], loadings['PC2'][i]))

plt.axhline(y=0, color='k', linestyle='--', alpha=0.3)
plt.axvline(x=0, color='k', linestyle='--', alpha=0.3)
plt.xlabel('Componente Principal 1')
plt.ylabel('Componente Principal 2')
plt.title('Círculo de Correlación')
plt.grid(True)

# Añadir un círculo unitario
theta = np.linspace(0, 2*np.pi, 100)
plt.plot(np.cos(theta), np.sin(theta), 'k--', alpha=0.3)
plt.axis('equal')
plt.show()

El círculo de correlación nos ayuda a interpretar las relaciones entre las características originales y los componentes principales. Las características que están cerca del borde del círculo y en direcciones similares están altamente correlacionadas entre sí.

t-SNE (t-Distributed Stochastic Neighbor Embedding)

t-SNE es un algoritmo de reducción de dimensionalidad no lineal que es particularmente adecuado para la visualización de datos de alta dimensión. A diferencia de PCA, que busca preservar la varianza global, t-SNE se centra en preservar las similitudes locales entre los puntos de datos.

Características Principales de t-SNE

  • No linealidad: Puede capturar estructuras no lineales en los datos.
  • Preservación de vecindarios locales: Los puntos que están cerca en el espacio original tenderán a estar cerca en el espacio reducido.
  • Hiperparámetros sensibles: La elección de la perplejidad y la tasa de aprendizaje puede afectar significativamente los resultados.

Implementación de t-SNE en Python

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.datasets import load_digits
from sklearn.manifold import TSNE
from sklearn.preprocessing import StandardScaler

# Cargar el conjunto de datos de dígitos escritos a mano
digits = load_digits()
X = digits.data
y = digits.target

# Estandarizar los datos
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Aplicar t-SNE
tsne = TSNE(n_components=2, random_state=42, perplexity=30, n_iter=1000)
X_tsne = tsne.fit_transform(X_scaled)

# Crear un DataFrame con los resultados
tsne_df = pd.DataFrame(data=X_tsne, columns=['Dimensión 1', 'Dimensión 2'])
tsne_df['Clase'] = y

# Visualizar los resultados
plt.figure(figsize=(10, 8))
for i in range(10):
    indices = tsne_df['Clase'] == i
    plt.scatter(tsne_df.loc[indices, 'Dimensión 1'],
                tsne_df.loc[indices, 'Dimensión 2'],
                label=str(i), s=50, alpha=0.7)

plt.title('Visualización t-SNE del Conjunto de Datos de Dígitos')
plt.xlabel('Dimensión 1')
plt.ylabel('Dimensión 2')
plt.legend(title='Dígito')
plt.grid(True, alpha=0.3)
plt.show()

# Comparar diferentes valores de perplejidad
perplexities = [5, 30, 50, 100]

plt.figure(figsize=(15, 10))
for i, perplexity in enumerate(perplexities, 1):
    # Aplicar t-SNE con diferente perplejidad
    tsne = TSNE(n_components=2, random_state=42, perplexity=perplexity, n_iter=1000)
    X_tsne = tsne.fit_transform(X_scaled)
    
    # Visualizar
    plt.subplot(2, 2, i)
    scatter = plt.scatter(X_tsne[:, 0], X_tsne[:, 1], c=y, cmap='tab10', alpha=0.7)
    plt.title(f't-SNE con perplejidad = {perplexity}')
    plt.xlabel('Dimensión 1')
    plt.ylabel('Dimensión 2')
    
plt.tight_layout()
plt.show()

Explicación del Código

  1. Carga de datos: Cargamos el conjunto de datos de dígitos escritos a mano, que contiene imágenes de 8x8 píxeles de dígitos del 0 al 9.
  2. Estandarización: Estandarizamos los datos para que tengan media 0 y desviación estándar 1.
  3. Aplicación de t-SNE: Creamos una instancia de t-SNE con 2 componentes y un valor de perplejidad de 30, luego ajustamos el modelo a nuestros datos.
  4. Visualización: Visualizamos los datos transformados, coloreando los puntos según la clase a la que pertenecen.
  5. Comparación de perplejidades: Mostramos cómo diferentes valores de perplejidad afectan a la visualización t-SNE.

La perplejidad es un parámetro que controla el equilibrio entre la atención que el algoritmo presta a la estructura local y global de los datos. Valores más bajos hacen que el algoritmo se centre más en la estructura local, mientras que valores más altos consideran más la estructura global.

Ventajas y Limitaciones de t-SNE

Ventajas

  • Excelente para visualizar estructuras no lineales
  • Preserva las relaciones locales entre puntos de datos
  • Efectivo para conjuntos de datos con muchas características
  • Útil para identificar agrupaciones (clústeres) en los datos

Limitaciones

  • Los resultados pueden variar con diferentes ejecuciones (debido a la inicialización aleatoria)
  • La elección de la perplejidad puede afectar significativamente los resultados
  • Computacionalmente costoso para conjuntos de datos grandes
  • Las distancias entre grupos en la visualización no son interpretables

UMAP (Uniform Manifold Approximation and Projection)

UMAP es un algoritmo de reducción de dimensionalidad que puede usarse para visualización similar a t-SNE, pero con un mejor rendimiento computacional y una mejor preservación de la estructura global de los datos.

Características Principales de UMAP

  • Eficiencia computacional: Generalmente más rápido que t-SNE, especialmente para conjuntos de datos grandes.
  • Preservación de la estructura global: Mejor que t-SNE para preservar tanto la estructura local como la global.
  • Hiperparámetros intuitivos: Los parámetros como `n_neighbors` y `min_dist` son más fáciles de interpretar que la perplejidad en t-SNE.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.datasets import fetch_openml
from sklearn.preprocessing import StandardScaler
import umap

# Cargar el conjunto de datos MNIST
print("Cargando el conjunto de datos MNIST...")
mnist = fetch_openml('mnist_784', as_frame=False)
X = mnist.data
y = mnist.target.astype(int)

# Tomar una muestra más pequeña para reducir el tiempo de cómputo
np.random.seed(42)
sample_size = 5000
indices = np.random.permutation(len(X))[:sample_size]
X_sample = X[indices]
y_sample = y[indices]

# Estandarizar los datos
print("Estandarizando los datos...")
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_sample)

# Aplicar UMAP
print("Aplicando UMAP...")
reducer = umap.UMAP(n_components=2, random_state=42, n_neighbors=15, min_dist=0.1)
X_umap = reducer.fit_transform(X_scaled)

# Crear un DataFrame con los resultados
umap_df = pd.DataFrame(data=X_umap, columns=['Dimensión 1', 'Dimensión 2'])
umap_df['Clase'] = y_sample

# Visualizar los resultados
plt.figure(figsize=(12, 10))
scatter = plt.scatter(X_umap[:, 0], X_umap[:, 1], c=y_sample, cmap='Spectral', s=5, alpha=0.7)
plt.colorbar(scatter, label='Dígito')
plt.title('Visualización UMAP del Conjunto de Datos MNIST (muestra de 5,000 dígitos)', fontsize=12)
plt.xlabel('Dimensión 1')
plt.ylabel('Dimensión 2')
plt.grid(True, alpha=0.2)
plt.show()

# Comparar diferentes valores de n_neighbors y min_dist
params_combinations = [
    {'n_neighbors': 5, 'min_dist': 0.1},
    {'n_neighbors': 50, 'min_dist': 0.1},
    {'n_neighbors': 15, 'min_dist': 0.01},
    {'n_neighbors': 15, 'min_dist': 0.5}
]

plt.figure(figsize=(15, 12))
for i, params in enumerate(params_combinations, 1):
    # Aplicar UMAP con diferentes parámetros
    reducer = umap.UMAP(n_components=2, random_state=42, 
                        n_neighbors=params['n_neighbors'], 
                        min_dist=params['min_dist'])
    X_umap = reducer.fit_transform(X_scaled)
    
    # Visualizar
    plt.subplot(2, 2, i)
    plt.scatter(X_umap[:, 0], X_umap[:, 1], c=y_sample, cmap='Spectral', s=5, alpha=0.7)
    plt.title(f"UMAP: n_neighbors={params['n_neighbors']}, min_dist={params['min_dist']}")
    plt.xlabel('Dimensión 1')
    plt.ylabel('Dimensión 2')
    
plt.tight_layout()
plt.show()

Explicación del Código

  1. Carga de datos: Cargamos el conjunto de datos MNIST, que contiene imágenes de dígitos escritos a mano de 28x28 píxeles.
  2. Muestreo: Tomamos una muestra más pequeña del conjunto de datos para reducir el tiempo de cómputo.
  3. Estandarización: Estandarizamos los datos para que tengan media 0 y desviación estándar 1.
  4. Aplicación de UMAP: Creamos una instancia de UMAP con 2 componentes y parámetros predeterminados, luego ajustamos el modelo a nuestros datos.
  5. Visualización: Visualizamos los datos transformados, coloreando los puntos según la clase a la que pertenecen.
  6. Comparación de parámetros: Mostramos cómo diferentes valores de `n_neighbors` y `min_dist` afectan a la visualización UMAP.

Los parámetros clave de UMAP son:

  • n_neighbors: Controla el equilibrio entre la estructura local y global. Valores más bajos se centran en la estructura local, mientras que valores más altos capturan mejor la estructura global.
  • min_dist: Controla la separación mínima entre puntos en el espacio de baja dimensión. Valores más bajos permiten una agrupación más densa de puntos similares.

Comparación entre PCA, t-SNE y UMAP

Característica PCA t-SNE UMAP
Tipo Lineal No lineal No lineal
Velocidad Muy rápida Lenta, especialmente para conjuntos grandes Rápida, incluso para conjuntos grandes
Estructura global Buena Mala Buena
Estructura local Mala para datos no lineales Excelente Muy buena
Interpretabilidad Alta (componentes lineales) Baja Baja
Uso recomendado Reducción inicial de dimensionalidad, visualización rápida Visualización de estructuras locales en conjuntos pequeños Visualización de conjuntos grandes, preservando estructura global y local

Para conjuntos de datos muy grandes, una estrategia común es usar primero PCA para reducir la dimensionalidad a un número manejable (por ejemplo, 50 componentes) y luego aplicar UMAP o t-SNE para la visualización final.

Aplicaciones Prácticas de la Reducción de Dimensionalidad

La reducción de dimensionalidad tiene numerosas aplicaciones en el mundo real. A continuación, se presentan algunos ejemplos prácticos:

1. Procesamiento de Imágenes

Las imágenes son datos de alta dimensión por naturaleza (cada píxel es una característica). La reducción de dimensionalidad permite:

  • Compresión de imágenes
  • Detección de características relevantes
  • Reconocimiento facial
  • Clasificación de imágenes médicas
Ejemplo: Reconocimiento Facial con PCA (Eigenfaces)
from sklearn.datasets import fetch_lfw_people
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt
import numpy as np

# Cargar el conjunto de datos de caras etiquetadas (Labeled Faces in the Wild)
lfw_people = fetch_lfw_people(min_faces_per_person=70, resize=0.4)
X = lfw_people.data
y = lfw_people.target

# Aplicar PCA para extraer las "eigenfaces"
n_components = 100  # Número de componentes principales a extraer
pca = PCA(n_components=n_components, whiten=True, random_state=42)
X_pca = pca.fit_transform(X)

# Visualizar las primeras eigenfaces
fig, axes = plt.subplots(2, 5, figsize=(15, 6),
                         subplot_kw={'xticks':[], 'yticks':[]},
                         gridspec_kw=dict(hspace=0.1, wspace=0.1))

for i, ax in enumerate(axes.flat):
    ax.imshow(pca.components_[i].reshape(50, 37), cmap='gray')
    ax.set_title(f'Componente {i+1}')

plt.suptitle('Eigenfaces - Componentes Principales', fontsize=16)
plt.show()

# Reconstruir las caras originales usando diferentes números de componentes
n_components_to_test = [10, 50, 100, 200, 400]

plt.figure(figsize=(15, 8))
for i, n in enumerate(n_components_to_test, 1):
    # Ajustar PCA con n componentes
    pca = PCA(n_components=n, whiten=True, random_state=42)
    X_pca = pca.fit_transform(X)
    
    # Reconstruir la imagen
    X_reconstructed = pca.inverse_transform(X_pca)
    
    # Mostrar la imagen original y reconstruida
    plt.subplot(2, len(n_components_to_test), i)
    plt.imshow(X[0].reshape(50, 37), cmap='gray')
    plt.title(f'Original\n({X.shape[1]} características)')
    plt.axis('off')
    
    plt.subplot(2, len(n_components_to_test), i + len(n_components_to_test))
    plt.imshow(X_reconstructed[0].reshape(50, 37), cmap='gray')
    plt.title(f'{n} componentes\n({n/X.shape[1]*100:.1f}% de varianza)')
    plt.axis('off')

plt.tight_layout()
plt.show()

2. Procesamiento de Lenguaje Natural (NLP)

En NLP, los textos se representan a menudo como vectores de alta dimensión (por ejemplo, TF-IDF o incrustaciones de palabras). La reducción de dimensionalidad permite:

  • Visualizar similitudes entre documentos
  • Reducir el ruido en los datos de texto
  • Mejorar el rendimiento de los modelos de clasificación
Ejemplo: Visualización de Documentos con t-SNE
from sklearn.datasets import fetch_20newsgroups
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.manifold import TSNE
import matplotlib.pyplot as plt
import numpy as np

# Cargar el conjunto de datos de noticias
categories = ['sci.space', 'rec.sport.baseball', 'comp.graphics', 'talk.politics.mideast']
newsgroups = fetch_20newsgroups(subset='train', categories=categories, remove=('headers', 'footers', 'quotes'))

# Convertir los textos a vectores TF-IDF
vectorizer = TfidfVectorizer(max_features=5000, stop_words='english')
X = vectorizer.fit_transform(newsgroups.data)

# Aplicar t-SNE para reducción a 2D
tsne = TSNE(n_components=2, random_state=42, perplexity=30, n_iter=1000)
X_tsne = tsne.fit_transform(X.toarray())

# Visualizar los resultados
plt.figure(figsize=(12, 8))
for i, category in enumerate(categories):
    indices = newsgroups.target == i
    plt.scatter(X_tsne[indices, 0], X_tsne[indices, 1], 
                label=category, alpha=0.7, s=50)

plt.title('Visualización de Documentos con t-SNE')
plt.xlabel('Dimensión 1')
plt.ylabel('Dimensión 2')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()

3. Genómica y Bioinformática

En genómica, los conjuntos de datos suelen tener miles o millones de características (genes) para un número relativamente pequeño de muestras. La reducción de dimensionalidad permite:

  • Identificar patrones en la expresión génica
  • Clasificar tipos de cáncer
  • Descubrir subtipos moleculares
Ejemplo: Análisis de Expresión Génica con UMAP
import scanpy as sc
import pandas as pd
import matplotlib.pyplot as plt

# Cargar un conjunto de datos de expresión génica (ejemplo con datos de células inmunes)
# Nota: Este es un ejemplo con datos simulados
# En la práctica, podrías cargar datos de un archivo .h5ad o .csv

# Datos simulados (en un caso real, cargarías tus propios datos)
# Por ejemplo: adata = sc.read_h5ad('tus_datos.h5ad')

# Para este ejemplo, usaremos un conjunto de datos de ejemplo de scanpy
adata = sc.datasets.pbmc3k()

# Preprocesamiento básico
sc.pp.filter_cells(adata, min_genes=200)
sc.pp.filter_genes(adata, min_cells=3)
adata.var['mt'] = adata.var_names.str.startswith('MT-')
sc.pp.calculate_qc_metrics(adata, qc_vars=['mt'], percent_top=None, log1p=False, inplace=True)

# Normalización
del adata.raw
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)
sc.pp.highly_variable_genes(adata, min_mean=0.0125, max_mean=3, min_disp=0.5)
adata = adata[:, adata.var.highly_variable]

# Escalar los datos
sc.pp.scale(adata, max_value=10)

# Aplicar PCA
sc.tl.pca(adata, svd_solver='arpack')

# Aplicar UMAP
sc.pp.neighbors(adata, n_neighbors=10, n_pcs=40)
sc.tl.umap(adata)

# Visualizar los resultados
plt.figure(figsize=(10, 8))
sc.pl.umap(adata, color='louvain', size=50, legend_loc='on data', show=False)
plt.title('UMAP de Expresión Génica (Células Inmunes)')
plt.show()

Ejercicio 1: Análisis de Componentes Principales (PCA)

Carga el conjunto de datos de vino de scikit-learn y realiza las siguientes tareas:

  1. Estandariza los datos.
  2. Aplica PCA para reducir la dimensionalidad a 2 componentes.
  3. Visualiza los datos en el espacio de los dos primeros componentes principales, coloreando los puntos según la clase de vino.
  4. ¿Qué porcentaje de la varianza total explican los dos primeros componentes principales?
  5. ¿Qué características originales contribuyen más a cada uno de los dos primeros componentes principales?

Puedes cargar el conjunto de datos de vino con from sklearn.datasets import load_wine. Usa pca.explained_variance_ratio_ para obtener la varianza explicada por cada componente.

from sklearn.datasets import load_wine
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt
import numpy as np

# Cargar el conjunto de datos de vino
wine = load_wine()
X = wine.data
y = wine.target
feature_names = wine.feature_names

# Estandarizar los datos
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Aplicar PCA
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)

# Visualizar los resultados
plt.figure(figsize=(10, 7))
for target, color in zip([0, 1, 2], ['r', 'g', 'b']):
    indices = y == target
    plt.scatter(X_pca[indices, 0], X_pca[indices, 1], 
                c=color, label=wine.target_names[target], alpha=0.7)

plt.xlabel(f'Componente Principal 1 ({pca.explained_variance_ratio_[0]:.1%})')
plt.ylabel(f'Componente Principal 2 ({pca.explained_variance_ratio_[1]:.1%})')
plt.title('PCA del Conjunto de Datos de Vino')
plt.legend()
plt.grid(True, alpha=0.3)
plt.show()

# Varianza explicada
print("Varianza explicada por cada componente:", pca.explained_variance_ratio_)
print("Varianza acumulada:", np.cumsum(pca.explained_variance_ratio_))

# Pesos de las características en los componentes principales
loadings = pd.DataFrame(pca.components_.T, 
                        columns=['PC1', 'PC2'],
                        index=feature_names)

print("\nPesos de las características en los componentes principales:")
print(loadings)

# Visualizar la importancia de las características
plt.figure(figsize=(10, 6))
plt.scatter(loadings['PC1'], loadings['PC2'])
for i, feature in enumerate(feature_names):
    plt.annotate(feature, (loadings['PC1'][i], loadings['PC2'][i]))

plt.axhline(y=0, color='k', linestyle='--', alpha=0.3)
plt.axvline(x=0, color='k', linestyle='--', alpha=0.3)
plt.xlabel('Componente Principal 1')
plt.ylabel('Componente Principal 2')
plt.title('Círculo de Correlación')
plt.grid(True, alpha=0.3)
plt.show()

Ejercicio 2: Comparación entre t-SNE y UMAP

Usando el conjunto de datos de dígitos de scikit-learn, realiza lo siguiente:

  1. Estandariza los datos.
  2. Aplica t-SNE para reducir la dimensionalidad a 2D.
  3. Aplica UMAP para reducir la dimensionalidad a 2D.
  4. Compara visualmente los resultados de ambos métodos.
  5. ¿Qué método crees que hace un mejor trabajo separando las diferentes clases? ¿Por qué?

Puedes cargar el conjunto de datos de dígitos con from sklearn.datasets import load_digits. Para UMAP, necesitarás instalar la biblioteca con pip install umap-learn.

from sklearn.datasets import load_digits
from sklearn.preprocessing import StandardScaler
from sklearn.manifold import TSNE
import umap
import matplotlib.pyplot as plt
import numpy as np

# Cargar el conjunto de datos de dígitos
digits = load_digits()
X = digits.data
y = digits.target

# Estandarizar los datos
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Aplicar t-SNE
tsne = TSNE(n_components=2, random_state=42, perplexity=30, n_iter=1000)
X_tsne = tsne.fit_transform(X_scaled)

# Aplicar UMAP
reducer = umap.UMAP(random_state=42, n_neighbors=15, min_dist=0.1)
X_umap = reducer.fit_transform(X_scaled)

# Visualizar los resultados
plt.figure(figsize=(18, 6))

# Gráfico t-SNE
plt.subplot(1, 2, 1)
scatter = plt.scatter(X_tsne[:, 0], X_tsne[:, 1], c=y, cmap='tab10', alpha=0.7)
plt.colorbar(scatter, label='Dígito')
plt.title('t-SNE de Dígitos (perplexity=30)')
plt.xlabel('Dimensión 1')
plt.ylabel('Dimensión 2')
plt.grid(True, alpha=0.3)

# Gráfico UMAP
plt.subplot(1, 2, 2)
scatter = plt.scatter(X_umap[:, 0], X_umap[:, 1], c=y, cmap='tab10', alpha=0.7)
plt.colorbar(scatter, label='Dígito')
plt.title('UMAP de Dígitos (n_neighbors=15, min_dist=0.1)')
plt.xlabel('Dimensión 1')
plt.ylabel('Dimensión 2')
plt.grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

# Comparación cualitativa
print("Comparación entre t-SNE y UMAP para el conjunto de datos de dígitos:")
print("\n1. t-SNE:")
print("- Excelente para preservar las distancias locales (puntos similares están cerca).")
print("- Puede perder la estructura global (distancias entre grupos no son significativas).")
print("- Los parámetros como 'perplexity' pueden afectar significativamente los resultados.")

print("\n2. UMAP:")
print("- Mejor preservación de la estructura global en comparación con t-SNE.")
print("- Generalmente más rápido que t-SNE para conjuntos de datos grandes.")
print("- Los parámetros 'n_neighbors' y 'min_dist' son más fáciles de interpretar que 'perplexity'.")

print("\nEn este caso, UMAP parece hacer un mejor trabajo separando las diferentes clases de dígitos, ") 
print("especialmente para dígitos que son estructuralmente similares (como 3, 5 y 8). Sin embargo, ") 
print("la elección entre t-SNE y UMAP puede depender del conjunto de datos específico y del objetivo del análisis.")

Ejercicio 3: Aplicación Práctica - Análisis de Sentimientos

En este ejercicio, utilizarás técnicas de reducción de dimensionalidad para analizar reseñas de películas del conjunto de datos de IMDB.

  1. Carga el conjunto de datos de reseñas de películas de IMDB.
  2. Convierte los textos a vectores numéricos usando TF-IDF.
  3. Reduce la dimensionalidad a 2D usando PCA, t-SNE y UMAP.
  4. Visualiza los resultados, coloreando los puntos según el sentimiento (positivo/negativo).
  5. ¿Qué método crees que hace un mejor trabajo separando las reseñas positivas de las negativas?

Puedes cargar el conjunto de datos de IMDB con from sklearn.datasets import fetch_20newsgroups o usar la API de la biblioteca nltk o tensorflow_datasets para obtener un conjunto de datos de reseñas de películas.

# Instalar las bibliotecas necesarias si no están instaladas
# !pip install umap-learn nltk

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.decomposition import PCA
from sklearn.manifold import TSNE
import umap
import nltk
from nltk.corpus import movie_reviews

# Descargar el conjunto de datos de reseñas de películas de NLTK
nltk.download('movie_reviews')

# Cargar los datos
documents = [(list(movie_reviews.words(fileid)), category)
             for category in movie_reviews.categories()
             for fileid in movie_reviews.fileids(category)]

# Convertir a formato de texto
X = [' '.join(words) for words, _ in documents]
y = [1 if category == 'pos' else 0 for _, category in documents]

# Tomar una muestra más pequeña para reducir el tiempo de cómputo
np.random.seed(42)
sample_size = 1000  # Reducir este número si el cómputo es demasiado lento
indices = np.random.permutation(len(X))[:sample_size]
X_sample = [X[i] for i in indices]
y_sample = [y[i] for i in indices]

# Convertir los textos a vectores TF-IDF
print("Convirtiendo textos a vectores TF-IDF...")
vectorizer = TfidfVectorizer(max_features=5000, stop_words='english')
X_tfidf = vectorizer.fit_transform(X_sample)

# Aplicar PCA
print("Aplicando PCA...")
pca = PCA(n_components=2, random_state=42)
X_pca = pca.fit_transform(X_tfidf.toarray())

# Aplicar t-SNE (puede llevar tiempo para conjuntos de datos grandes)
print("Aplicando t-SNE (esto puede tardar unos minutos)...")
tsne = TSNE(n_components=2, random_state=42, perplexity=30, n_iter=1000)
X_tsne = tsne.fit_transform(X_tfidf.toarray())

# Aplicar UMAP
print("Aplicando UMAP...")
reducer = umap.UMAP(random_state=42, n_neighbors=15, min_dist=0.1)
X_umap = reducer.fit_transform(X_tfidf)

# Visualizar los resultados
plt.figure(figsize=(18, 5))

# Gráfico PCA
plt.subplot(1, 3, 1)
for label in [0, 1]:
    indices = np.array(y_sample) == label
    plt.scatter(X_pca[indices, 0], X_pca[indices, 1], 
                label='Positivo' if label == 1 else 'Negativo',
                alpha=0.6, s=30)
plt.title(f'PCA (Varianza: {np.sum(pca.explained_variance_ratio_):.1%})')
plt.xlabel('Componente Principal 1')
plt.ylabel('Componente Principal 2')
plt.legend()
plt.grid(True, alpha=0.3)

# Gráfico t-SNE
plt.subplot(1, 3, 2)
for label in [0, 1]:
    indices = np.array(y_sample) == label
    plt.scatter(X_tsne[indices, 0], X_tsne[indices, 1], 
                label='Positivo' if label == 1 else 'Negativo',
                alpha=0.6, s=30)
plt.title('t-SNE (perplexity=30)')
plt.xlabel('Dimensión 1')
plt.ylabel('Dimensión 2')
plt.legend()
plt.grid(True, alpha=0.3)

# Gráfico UMAP
plt.subplot(1, 3, 3)
for label in [0, 1]:
    indices = np.array(y_sample) == label
    plt.scatter(X_umap[indices, 0], X_umap[indices, 1], 
                label='Positivo' if label == 1 else 'Negativo',
                alpha=0.6, s=30)
plt.title('UMAP (n_neighbors=15, min_dist=0.1)')
plt.xlabel('Dimensión 1')
plt.ylabel('Dimensión 2')
plt.legend()
plt.grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

# Análisis de los resultados
print("\nAnálisis de los resultados:")
print("1. PCA: Aunque es rápido, PCA (lineal) tiene dificultades para separar claramente las reseñas positivas de las negativas.")
print("2. t-SNE: Mejora la separación, pero puede crear agrupaciones artificiales y es sensible a la perplejidad.")
print("3. UMAP: Proporciona una buena separación entre las clases y es más rápido que t-SNE para conjuntos de datos grandes.")

print("\nConclusión: Para este conjunto de datos de análisis de sentimientos, UMAP parece ofrecer la mejor separación ") 
print("entre reseñas positivas y negativas, aunque la elección final del método puede depender de los requisitos ") 
print("específicos del problema, como la necesidad de velocidad, interpretabilidad o la preservación de estructuras específicas.")

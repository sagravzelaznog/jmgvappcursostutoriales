3.6: Árboles de Decisión

Curso de Inteligencia Artificial - Módulo 3: Machine Learning

Introducción a los Árboles de Decisión

Los árboles de decisión son uno de los algoritmos de aprendizaje supervisado más intuitivos y ampliamente utilizados en el aprendizaje automático. Se pueden utilizar tanto para problemas de clasificación como de regresión, y son la base para algoritmos más avanzados como Random Forest y Gradient Boosting.

Estructura de un Árbol de Decisión

Un árbol de decisión es una estructura jerárquica que consta de:

  • Nodos raíz y nodos internos: Representan pruebas sobre las características
  • Ramas: Representan el resultado de las pruebas
  • Nodos hoja: Representan las predicciones o decisiones finales

La principal ventaja de los árboles de decisión es que son fáciles de entender e interpretar, ya que pueden visualizarse como un diagrama de flujo donde cada nodo interno representa una "pregunta" sobre una característica, cada rama representa el resultado de la pregunta y cada nodo hoja representa una etiqueta de clase o un valor de regresión.

Construcción de Árboles de Decisión

La construcción de un árbol de decisión implica dividir recursivamente el espacio de características en regiones más pequeñas. Los algoritmos de árboles de decisión suelen utilizar un enfoque codicioso para dividir el espacio de características de manera que maximicen la homogeneidad de los nodos resultantes.

Métricas de División

1. Para Clasificación:

Índice Gini

Mide la impureza de un nodo. Un nodo puro (todos los elementos son de la misma clase) tiene un índice de Gini de 0.

Gini = 1 - Σ(pᵢ)²

Gini

Donde pᵢ es la proporción de elementos de la clase i en el nodo.

Entropía y Ganancia de Información

La entropía mide la incertidumbre en un conjunto de datos. La ganancia de información es la reducción en la entropía.

Entropía = -Σ(pᵢ * log₂(pᵢ))

Ganancia de Información = Entropía(padre) - Σ((nᵢ/n) * Entropía(hijoᵢ))

Entropía
Error de Clasificación

Proporción de ejemplos mal clasificados en un nodo.

Error = 1 - max(pᵢ)

2. Para Regresión:

Error Cuadrático Medio (MSE)

Minimiza la varianza dentro de cada nodo hoja.

MSE = (1/n) * Σ(yᵢ - ȳ)²

Error Absoluto Medio (MAE)

Menos sensible a valores atípicos que MSE.

MAE = (1/n) * Σ|yᵢ - mediana(y)|

Algoritmo de Construcción

El algoritmo básico para construir un árbol de decisión es el siguiente:

  1. Comienza con todo el conjunto de entrenamiento en el nodo raíz.
  2. Para cada característica, encuentra el mejor punto de división que maximice la métrica de división seleccionada.
  3. Divide el nodo en subnodos utilizando la mejor división encontrada.
  4. Repite los pasos 2-3 recursivamente para cada subnodo hasta que se cumpla un criterio de parada.
  5. Asigna a cada nodo hoja la clase mayoritaria (clasificación) o el valor medio (regresión) de las muestras en ese nodo.

Hiperparámetros de los Árboles de Decisión

Los árboles de decisión tienen varios hiperparámetros que controlan su crecimiento y complejidad. La selección adecuada de estos parámetros es crucial para evitar el sobreajuste.

1. Control de Profundidad

  • max_depth: Profundidad máxima del árbol. Árboles más profundos pueden capturar patrones más complejos pero son propensos a sobreajuste.
  • min_samples_split: Número mínimo de muestras requeridas para dividir un nodo interno.
  • min_samples_leaf: Número mínimo de muestras requeridas en un nodo hoja.

2. Criterios de División

  • criterion: Función para medir la calidad de una división ('gini', 'entropy' para clasificación; 'mse', 'mae' para regresión).
  • max_features: Número de características a considerar al buscar la mejor división.
  • splitter: Estrategia para elegir la división en cada nodo ('best' o 'random').

3. Control de Poda

  • min_impurity_decrease: Un nodo se dividirá si esta división induce una disminución de la impureza mayor o igual a este valor.
  • ccp_alpha: Parámetro de complejidad para la poda de coste-complejidad.

Implementación en Python

A continuación, veremos cómo implementar árboles de decisión en Python utilizando las bibliotecas más comunes como NumPy, Pandas, Matplotlib y Scikit-learn.

1. Clasificación con Árboles de Decisión

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.datasets import load_iris
from sklearn.tree import DecisionTreeClassifier, plot_tree, export_text
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, classification_report, confusion_matrix
import seaborn as sns

# Cargar el conjunto de datos de Iris
iris = load_iris()
X = pd.DataFrame(iris.data, columns=iris.feature_names)
y = iris.target

# Dividir los datos en conjuntos de entrenamiento y prueba
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42, stratify=y
)

# Crear y entrenar el modelo de árbol de decisión
tree_clf = DecisionTreeClassifier(
    max_depth=3,
    criterion='gini',
    min_samples_split=2,
    min_samples_leaf=1,
    random_state=42
)
tree_clf.fit(X_train, y_train)

# Hacer predicciones
y_pred = tree_clf.predict(X_test)
y_pred_proba = tree_clf.predict_proba(X_test)

# Evaluar el modelo
accuracy = accuracy_score(y_test, y_pred)
print(f"Precisión del modelo: {accuracy:.4f}")

# Mostrar matriz de confusión
cm = confusion_matrix(y_test, y_pred)
plt.figure(figsize=(8, 6))
sns.heatmap(cm, annot=True, fmt='d', cmap='Blues', 
            xticklabels=iris.target_names, 
            yticklabels=iris.target_names)
plt.xlabel('Predicho')
plt.ylabel('Real')
plt.title('Matriz de Confusión')
plt.show()

# Mostrar reporte de clasificación
print("\nReporte de Clasificación:")
print(classification_report(y_test, y_pred, target_names=iris.target_names))

# Visualizar el árbol de decisión
plt.figure(figsize=(20, 10))
plot_tree(
    tree_clf,
    feature_names=iris.feature_names,
    class_names=iris.target_names,
    filled=True,
    rounded=True,
    fontsize=10
)
plt.title("Árbol de Decisión para Clasificación de Flores Iris")
plt.show()

# Mostrar las reglas del árbol
tree_rules = export_text(
    tree_clf,
    feature_names=list(iris.feature_names),
    show_weights=True
)
print("\nReglas del Árbol de Decisión:")
print(tree_rules)

# Importancia de las características
feature_importance = pd.DataFrame({
    'Característica': iris.feature_names,
    'Importancia': tree_clf.feature_importances_
}).sort_values('Importancia', ascending=False)

plt.figure(figsize=(10, 6))
plt.barh(feature_importance['Característica'], feature_importance['Importancia'])
plt.xlabel('Importancia')
plt.title('Importancia de las Características')
plt.tight_layout()
plt.show()

Ventajas y Desventajas

Ventajas

  • Fácil de entender e interpretar: Los árboles de decisión pueden visualizarse y son fáciles de explicar a personas sin conocimientos técnicos.
  • Poco preprocesamiento de datos: No requieren normalización de características y pueden manejar tanto características numéricas como categóricas.
  • Manejo de relaciones no lineales: Pueden capturar interacciones no lineales entre características.
  • Robustos a valores atípicos: Los valores atípicos tienen menos impacto en los árboles de decisión que en otros algoritmos.
  • Selección de características implícita: Los árboles de decisión realizan automáticamente selección de características.

Desventajas

  • Propensos al sobreajuste: Pueden crear árboles demasiado complejos que no generalizan bien a datos no vistos.
  • Inestables: Pequeños cambios en los datos pueden resultar en un árbol completamente diferente.
  • Sesgo hacia características con más niveles: Los atributos con más niveles tienen más probabilidades de ser seleccionados.
  • Dificultad para capturar relaciones aditivas: No son buenos para aprender relaciones aditivas entre características.
  • Óptimos locales: El algoritmo de construcción del árbol es codicioso y puede quedarse atascado en óptimos locales.

Ejercicio 1: Clasificación con Árboles de Decisión

Utiliza el conjunto de datos de vino de Scikit-learn para implementar un clasificador de árbol de decisión que prediga la clase de vino basándose en sus características químicas. Sigue estos pasos:

  1. Carga el conjunto de datos de vino
  2. Explora y preprocesa los datos
  3. Divide los datos en conjuntos de entrenamiento y prueba
  4. Entrena un árbol de decisión con diferentes hiperparámetros
  5. Evalúa el rendimiento usando precisión, matriz de confusión y reporte de clasificación
  6. Visualiza el árbol de decisión y la importancia de las características
  7. Optimiza los hiperparámetros usando GridSearchCV

Puedes cargar el conjunto de datos de vino usando from sklearn.datasets import load_wine. Este conjunto de datos contiene 13 características químicas de vinos de tres cultivares diferentes en la región de Italia.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.datasets import load_wine
from sklearn.tree import DecisionTreeClassifier, plot_tree, export_text
from sklearn.model_selection import train_test_split, GridSearchCV
from sklearn.metrics import accuracy_score, classification_report, confusion_matrix
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline

# 1. Cargar el conjunto de datos de vino
wine = load_wine()
X = pd.DataFrame(wine.data, columns=wine.feature_names)
y = wine.target

# 2. Explorar los datos
print("Forma de X:", X.shape)
print("\nPrimeras filas de X:")
print(X.head())

# Visualizar la distribución de las clases
plt.figure(figsize=(8, 5))
sns.countplot(x=y)
plt.xticks(ticks=[0, 1, 2], labels=wine.target_names)
plt.xlabel('Clase de Vino')
plt.ylabel('Cantidad')
plt.title('Distribución de Clases')
plt.show()

# Visualizar la correlación entre características
plt.figure(figsize=(12, 8))
sns.heatmap(X.corr(), annot=True, cmap='coolwarm', fmt='.2f')
plt.title('Matriz de Correlación')
plt.tight_layout()
plt.show()

# 3. Dividir los datos en conjuntos de entrenamiento y prueba
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42, stratify=y
)

# 4. Crear un pipeline con escalado y árbol de decisión
pipeline = Pipeline([
    ('scaler', StandardScaler()),
    ('tree', DecisionTreeClassifier(random_state=42))
])

# 5. Definir la cuadrícula de parámetros para la búsqueda
param_grid = {
    'tree__criterion': ['gini', 'entropy'],
    'tree__max_depth': [None, 3, 5, 7, 10],
    'tree__min_samples_split': [2, 5, 10],
    'tree__min_samples_leaf': [1, 2, 4],
    'tree__max_features': ['sqrt', 'log2', None]
}

# 6. Configurar la búsqueda en cuadrícula con validación cruzada
grid_search = GridSearchCV(
    estimator=pipeline,
    param_grid=param_grid,
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    verbose=1
)

# 7. Realizar la búsqueda en la cuadrícula
print("\nIniciando la búsqueda en cuadrícula...")
grid_search.fit(X_train, y_train)

# 8. Mostrar los mejores parámetros
print("\nMejores parámetros:", grid_search.best_params_)
print("Mejor puntuación de validación cruzada: {:.4f}".format(grid_search.best_score_))

# 9. Evaluar el mejor modelo en el conjunto de prueba
best_model = grid_search.best_estimator_
y_pred = best_model.predict(X_test)
y_pred_proba = best_model.predict_proba(X_test)

test_accuracy = accuracy_score(y_test, y_pred)
print("\nPrecisión en el conjunto de prueba: {:.4f}".format(test_accuracy))

# 10. Mostrar matriz de confusión
cm = confusion_matrix(y_test, y_pred)
plt.figure(figsize=(8, 6))
sns.heatmap(cm, annot=True, fmt='d', cmap='Blues', 
            xticklabels=wine.target_names, 
            yticklabels=wine.target_names)
plt.xlabel('Predicho')
plt.ylabel('Real')
plt.title('Matriz de Confusión')
plt.show()

# 11. Mostrar reporte de clasificación
print("\nReporte de Clasificación:")
print(classification_report(y_test, y_pred, target_names=wine.target_names))

# 12. Visualizar el árbol de decisión
plt.figure(figsize=(20, 10))
plot_tree(
    best_model.named_steps['tree'],
    feature_names=wine.feature_names,
    class_names=wine.target_names,
    filled=True,
    rounded=True,
    fontsize=10
)
plt.title("Mejor Árbol de Decisión para Clasificación de Vinos")
plt.show()

# 13. Mostrar las reglas del árbol
tree_rules = export_text(
    best_model.named_steps['tree'],
    feature_names=list(wine.feature_names),
    show_weights=True
)
print("\nReglas del Árbol de Decisión (primeras 20 líneas):")
print('\n'.join(tree_rules.split('\n')[:20]))  # Mostrar solo las primeras 20 líneas

# 14. Importancia de las características
feature_importance = pd.DataFrame({
    'Característica': wine.feature_names,
    'Importancia': best_model.named_steps['tree'].feature_importances_
}).sort_values('Importancia', ascending=False)

plt.figure(figsize=(10, 6))
plt.barh(feature_importance['Característica'], feature_importance['Importancia'])
plt.xlabel('Importancia')
plt.title('Importancia de las Características')
plt.tight_layout()
plt.show()

Ejercicio 2: Regresión con Árboles de Decisión

Utiliza el conjunto de datos de precios de vino tinto para implementar un modelo de regresión con árboles de decisión que prediga la calidad del vino basándose en sus características químicas. Sigue estos pasos:

  1. Cargar y explorar los datos de vino tinto
  2. Realizar análisis exploratorio de datos (EDA)
  3. Preprocesar los datos (escalado, manejo de valores atípicos)
  4. Dividir los datos en conjuntos de entrenamiento y prueba
  5. Entrenar un árbol de regresión con diferentes hiperparámetros
  6. Evaluar el rendimiento usando MSE, R² y visualizaciones
  7. Optimizar los hiperparámetros usando búsqueda en cuadrícula
  8. Analizar la importancia de las características

Puedes cargar el conjunto de datos de vino tinto desde el repositorio UCI Machine Learning usando pandas: df = pd.read_csv('https://archive.ics.uci.edu/ml/machine-learning-databases/wine-quality/winequality-red.csv', sep=';'). Este conjunto de datos contiene 11 características fisicoquímicas y una variable objetivo (quality) que varía de 0 a 10.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.tree import DecisionTreeRegressor, plot_tree, export_text
from sklearn.model_selection import train_test_split, GridSearchCV
from sklearn.metrics import mean_squared_error, r2_score, mean_absolute_error
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline

# 1. Cargar el conjunto de datos de vino tinto
url = 'https://archive.ics.uci.edu/ml/machine-learning-databases/wine-quality/winequality-red.csv'
df = pd.read_csv(url, sep=';')

# 2. Explorar los datos
print("Forma del conjunto de datos:", df.shape)
print("\nPrimeras filas del conjunto de datos:")
print(df.head())

# Ver valores faltantes
print("\nValores faltantes por columna:")
print(df.isnull().sum())

# Estadísticas descriptivas
print("\nEstadísticas descriptivas:")
print(df.describe())

# 3. Análisis exploratorio de datos (EDA)
# Distribución de la variable objetivo (calidad)
plt.figure(figsize=(10, 5))
sns.countplot(x='quality', data=df)
plt.title('Distribución de la Calidad del Vino')
plt.xlabel('Calidad (0-10)')
plt.ylabel('Cantidad')
plt.show()

# Matriz de correlación
plt.figure(figsize=(12, 8))
correlation_matrix = df.corr()
sns.heatmap(correlation_matrix, annot=True, cmap='coolwarm', fmt='.2f', linewidths=0.5)
plt.title('Matriz de Correlación')
plt.tight_layout()
plt.show()

# 4. Preparar los datos para el modelo
X = df.drop('quality', axis=1)
y = df['quality']

# 5. Dividir los datos en conjuntos de entrenamiento y prueba
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42
)

# 6. Crear un pipeline con escalado y árbol de regresión
pipeline = Pipeline([
    ('scaler', StandardScaler()),
    ('tree', DecisionTreeRegressor(random_state=42))
])

# 7. Definir la cuadrícula de parámetros para la búsqueda
param_grid = {
    'tree__criterion': ['mse', 'friedman_mse', 'mae'],
    'tree__max_depth': [None, 3, 5, 7, 10],
    'tree__min_samples_split': [2, 5, 10],
    'tree__min_samples_leaf': [1, 2, 4],
    'tree__max_features': ['auto', 'sqrt', 'log2', None],
    'tree__min_impurity_decrease': [0.0, 0.01, 0.05, 0.1]
}

# 8. Configurar la búsqueda en cuadrícula con validación cruzada
grid_search = GridSearchCV(
    estimator=pipeline,
    param_grid=param_grid,
    cv=5,
    scoring='neg_mean_squared_error',
    n_jobs=-1,
    verbose=1
)

# 9. Realizar la búsqueda en la cuadrícula
print("\nIniciando la búsqueda en cuadrícula...")
grid_search.fit(X_train, y_train)

# 10. Mostrar los mejores parámetros
print("\nMejores parámetros:", grid_search.best_params_)
print("Mejor puntuación de validación cruzada (MSE negativo): {:.4f}".format(grid_search.best_score_))

# 11. Evaluar el mejor modelo en el conjunto de prueba
best_model = grid_search.best_estimator_
y_pred = best_model.predict(X_test)

# Calcular métricas de rendimiento
mse = mean_squared_error(y_test, y_pred)
mae = mean_absolute_error(y_test, y_pred)
r2 = r2_score(y_test, y_pred)

print("\nMétricas de rendimiento en el conjunto de prueba:")
print(f"Error Cuadrático Medio (MSE): {mse:.4f}")
print(f"Error Absoluto Medio (MAE): {mae:.4f}")
print(f"Coeficiente de Determinación (R²): {r2:.4f}")

# 12. Visualizar predicciones vs valores reales
plt.figure(figsize=(10, 6))
plt.scatter(y_test, y_pred, alpha=0.5)
plt.plot([y_test.min(), y_test.max()], [y_test.min(), y_test.max()], 'r--', lw=2)
plt.xlabel('Valores Reales')
plt.ylabel('Predicciones')
plt.title('Valores Reales vs Predicciones')
plt.grid(True)
plt.show()

# 13. Visualizar el árbol de decisión (solo si la profundidad no es muy grande)
if grid_search.best_params_['tree__max_depth'] is not None and grid_search.best_params_['tree__max_depth'] <= 5:
    plt.figure(figsize=(20, 10))
    plot_tree(
        best_model.named_steps['tree'],
        feature_names=X.columns,
        filled=True,
        rounded=True,
        fontsize=10
    )
    plt.title("Árbol de Decisión para Regresión de Calidad de Vino")
    plt.show()
else:
    print("\nEl árbol es demasiado grande para visualizarlo. La profundidad máxima es:", 
          grid_search.best_params_['tree__max_depth'])

# 14. Importancia de las características
feature_importance = pd.DataFrame({
    'Característica': X.columns,
    'Importancia': best_model.named_steps['tree'].feature_importances_
}).sort_values('Importancia', ascending=False)

plt.figure(figsize=(10, 6))
plt.barh(feature_importance['Característica'], feature_importance['Importancia'])
plt.xlabel('Importancia')
plt.title('Importancia de las Características')
plt.tight_layout()
plt.show()

# 15. Visualizar la importancia de las características en orden descendente
plt.figure(figsize=(10, 6))
sns.barplot(x='Importancia', y='Característica', data=feature_importance.sort_values('Importancia', ascending=False))
plt.title('Importancia de las Características')
plt.tight_layout()
plt.show()

# 16. Visualizar la relación entre las características más importantes y la calidad
if not feature_importance.empty:
    top_features = feature_importance.head(3)['Característica'].values
    
    plt.figure(figsize=(15, 5))
    for i, feature in enumerate(top_features, 1):
        plt.subplot(1, 3, i)
        sns.regplot(x=df[feature], y=df['quality'], scatter_kws={'alpha':0.3})
        plt.title(f'Relación entre {feature} y Calidad')
        plt.xlabel(feature)
        plt.ylabel('Calidad')
    
    plt.tight_layout()
    plt.show()

Los árboles de decisión son especialmente útiles cuando se necesita un modelo interpretable que pueda manejar tanto características numéricas como categóricas. Sin embargo, es importante recordar sus limitaciones, especialmente su tendencia al sobreajuste, que puede mitigarse mediante técnicas de poda o utilizando métodos de conjunto como Random Forest o Gradient Boosting.

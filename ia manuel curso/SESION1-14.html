Introducción a las Pruebas Unitarias

Las pruebas unitarias son una práctica fundamental en el desarrollo de software que consiste en verificar que cada unidad de código (funciones, métodos, clases) funcione correctamente de forma aislada. En Python, el módulo unittest proporciona un marco de trabajo para escribir y ejecutar pruebas.

¿Por qué son importantes las pruebas unitarias?

  • Detección temprana de errores: Permiten identificar problemas en las primeras etapas del desarrollo.
  • Documentación viva: Sirven como documentación del comportamiento esperado del código.
  • Facilitan el mantenimiento: Permiten realizar cambios con confianza, sabiendo que las pruebas detectarán regresiones.
  • Mejoran el diseño: Escribir código que sea fácil de probar generalmente conduce a un mejor diseño.

Estructura básica de una prueba unitaria

Una prueba unitaria típica sigue el patrón "Arrange-Act-Assert" (Organizar-Actuar-Afirmar):

Python 
import unittest

# Código a probar
def suma(a, b):
    return a + b

# Clase de prueba
class TestSuma(unittest.TestCase):
    def test_suma_positivos(self):
        # Organizar (Arrange)
        a = 5
        b = 3
        
        # Actuar (Act)
        resultado = suma(a, b)
        
        # Afirmar (Assert)
        self.assertEqual(resultado, 8, "La suma de 5 + 3 debe ser 8")

    def test_suma_negativos(self):
        self.assertEqual(suma(-1, -1), -2, "La suma de -1 + (-1) debe ser -2")

    def test_suma_mezclada(self):
        self.assertEqual(suma(10, -5), 5, "La suma de 10 + (-5) debe ser 5")

if __name__ == '__main__':
    unittest.main()

Para ejecutar las pruebas, guarda el código en un archivo (por ejemplo, test_suma.py) y ejecuta:

python -m unittest test_suma.py
Consejo: Puedes ejecutar una prueba específica con python -m unittest test_suma.TestSuma.test_suma_positivos o usar el flag -v para obtener más detalles: python -m unittest -v test_suma.py

Aserciones Comunes

El módulo unittest proporciona varios métodos de aserción. Aquí están los más utilizados:

Método Comprueba que Ejemplo
assertEqual(a, b) a == b self.assertEqual(suma(2,2), 4)
assertNotEqual(a, b) a != b self.assertNotEqual(func(), None)
assertTrue(x) bool(x) es True self.assertTrue(es_par(4))
assertFalse(x) bool(x) es False self.assertFalse(es_par(3))
assertIs(a, b) a es b self.assertIs(obj, None)
assertIsNot(a, b) a no es b self.assertIsNot(result, None)
assertIsNone(x) x es None self.assertIsNone(resultado)
assertIn(a, b) a en b self.assertIn(3, [1, 2, 3])
assertNotIn(a, b) a no está en b self.assertNotIn(4, [1, 2, 3])
assertRaises(exc, fun, *args, **kwds) fun(*args, **kwds) lanza exc with self.assertRaises(ValueError): int('no_es_un_numero')

Pruebas Parametrizadas

Las pruebas parametrizadas permiten ejecutar la misma lógica de prueba con diferentes conjuntos de datos. Aunque unittest no tiene soporte nativo para pruebas parametrizadas, podemos usar el decorador @parameterized de la biblioteca parameterized.

Primero, instala la biblioteca:

pip install parameterized
Python 
import unittest
from parameterized import parameterized

def es_par(numero):
    """Devuelve True si el número es par, False en caso contrario."""
    if not isinstance(numero, int):
        raise TypeError("El argumento debe ser un entero")
    return numero % 2 == 0

class TestEsParParametrizado(unittest.TestCase):
    @parameterized.expand([
        (2, True),
        (3, False),
        (0, True),
        (-2, True),
        (-3, False),
    ])
    def test_es_par(self, numero, esperado):
        self.assertEqual(es_par(numero), esperado, 
                        f"es_par({numero}) debería ser {esperado}")
    
    def test_tipo_no_entero(self):
        with self.assertRaises(TypeError):
            es_par("no es un número")

if __name__ == '__main__':
    unittest.main()

Fixtures (Preparación y Limpieza)

Los fixtures permiten configurar un entorno conocido antes de cada prueba y limpiarlo después. Los métodos más utilizados son setUp() y tearDown().

Python 
import unittest
import os
import tempfile

class TestArchivos(unittest.TestCase):
    def setUp(self):
        # Se ejecuta antes de cada prueba
        self.directorio_temporal = tempfile.mkdtemp()
        self.archivo_temporal = os.path.join(self.directorio_temporal, 'datos.txt')
        with open(self.archivo_temporal, 'w') as f:
            f.write("Hola, mundo!")
    
    def tearDown(self):
        # Se ejecuta después de cada prueba
        if os.path.exists(self.archivo_temporal):
            os.remove(self.archivo_temporal)
        if os.path.exists(self.directorio_temporal):
            os.rmdir(self.directorio_temporal)
    
    def test_archivo_existe(self):
        self.assertTrue(os.path.exists(self.archivo_temporal))
    
    def test_contenido_archivo(self):
        with open(self.archivo_temporal, 'r') as f:
            contenido = f.read()
        self.assertEqual(contenido, "Hola, mundo!")

if __name__ == '__main__':
    unittest.main()

Métodos de Configuración

  • setUp(): Se ejecuta antes de cada método de prueba.
  • tearDown(): Se ejecuta después de cada método de prueba.
  • setUpClass(): Método de clase que se ejecuta una vez antes de todas las pruebas de la clase.
  • tearDownClass(): Método de clase que se ejecuta una vez después de todas las pruebas de la clase.

Mocks y Parches

Los mocks son objetos simulados que imitan el comportamiento de objetos reales de manera controlada. Son útiles para aislar el código bajo prueba de sus dependencias externas.

Python 
import unittest
from unittest.mock import patch, MagicMock

# Código a probar
def obtener_datos_desde_api(url):
    import requests
    response = requests.get(url)
    return response.json()

# Clase de prueba
class TestAPIMocking(unittest.TestCase):
    @patch('requests.get')
    def test_obtener_datos_desde_api(self, mock_get):
        # Configurar el mock
        mock_response = MagicMock()
        mock_response.json.return_value = {"clave": "valor"}
        mock_get.return_value = mock_response
        
        # Ejecutar el código bajo prueba
        resultado = obtener_datos_desde_api("https://api.ejemplo.com/datos")
        
        # Verificar que se llamó a requests.get con la URL correcta
        mock_get.assert_called_once_with("https://api.ejemplo.com/datos")
        
        # Verificar el resultado
        self.assertEqual(resultado, {"clave": "valor"})

if __name__ == '__main__':
    unittest.main()
Consejo: Usa @patch para reemplazar objetos en el espacio de nombres del módulo que estás probando. El mock se pasa como argumento a la función de prueba.

Cobertura de Código

La cobertura de código mica qué parte de tu código está siendo ejercitada por las pruebas. Para medir la cobertura, puedes usar la biblioteca coverage.py.

Primero, instala la biblioteca:

pip install coverage

Supongamos que tienes un módulo calculadora.py:

Python 
# calculadora.py
def suma(a, b):
    return a + b

def resta(a, b):
    return a - b

def multiplicacion(a, b):
    return a * b

def division(a, b):
    if b == 0:
        raise ValueError("No se puede dividir por cero")
    return a / b

Y un archivo de prueba test_calculadora.py:

Python 
# test_calculadora.py
import unittest
from calculadora import suma, division

class TestCalculadora(unittest.TestCase):
    def test_suma(self):
        self.assertEqual(suma(2, 3), 5)
        self.assertEqual(suma(-1, 1), 0)
    
    def test_division(self):
        self.assertEqual(division(10, 2), 5)
        with self.assertRaises(ValueError):
            division(10, 0)

if __name__ == '__main__':
    unittest.main()

Para ejecutar las pruebas con cobertura:

coverage run -m unittest test_calculadora.py

Para ver el informe de cobertura:

coverage report -m

O para un informe HTML más detallado:

coverage html

Esto generará un directorio htmlcov/ con un informe HTML que puedes abrir en tu navegador.

Ejercicios Prácticos

Ejercicio 1: Pruebas para una calculadora científica

Crea una clase CalculadoraCientifica con los siguientes métodos y escribe pruebas unitarias para cada uno:

  • raiz_cuadrada(x): Devuelve la raíz cuadrada de x
  • potencia(base, exponente): Devuelve base elevada a exponente
  • factorial(n): Devuelve el factorial de n (n!)
  • es_primo(n): Devuelve True si n es primo, False en caso contrario

Recuerda probar casos límite como números negativos, cero y uno. Para el método es_primo, asegúrate de probar con números primos y no primos.

Python 
import math
import unittest

class CalculadoraCientifica:
    @staticmethod
    def raiz_cuadrada(x):
        if x < 0:
            raise ValueError("No se puede calcular la raíz cuadrada de un número negativo")
        return math.sqrt(x)
    
    @staticmethod
    def potencia(base, exponente):
        return base ** exponente
    
    @staticmethod
    def factorial(n):
        if not isinstance(n, int) or n < 0:
            raise ValueError("El factorial solo está definido para enteros no negativos")
        if n == 0 or n == 1:
            return 1
        return n * CalculadoraCientifica.factorial(n - 1)
    
    @staticmethod
    def es_primo(n):
        if n <= 1:
            return False
        if n == 2:
            return True
        if n % 2 == 0:
            return False
        for i in range(3, int(math.sqrt(n)) + 1, 2):
            if n % i == 0:
                return False
        return True

class TestCalculadoraCientifica(unittest.TestCase):
    def setUp(self):
        self.calc = CalculadoraCientifica()
    
    def test_raiz_cuadrada(self):
        self.assertAlmostEqual(self.calc.raiz_cuadrada(16), 4.0)
        self.assertAlmostEqual(self.calc.raiz_cuadrada(0), 0.0)
        with self.assertRaises(ValueError):
            self.calc.raiz_cuadrada(-1)
    
    def test_potencia(self):
        self.assertEqual(self.calc.potencia(2, 3), 8)
        self.assertEqual(self.calc.potencia(5, 0), 1)
        self.assertAlmostEqual(self.calc.potencia(4, -1), 0.25)
    
    def test_factorial(self):
        self.assertEqual(self.calc.factorial(0), 1)
        self.assertEqual(self.calc.factorial(1), 1)
        self.assertEqual(self.calc.factorial(5), 120)
        with self.assertRaises(ValueError):
            self.calc.factorial(-1)
        with self.assertRaises(ValueError):
            self.calc.factorial(2.5)
    
    def test_es_primo(self):
        self.assertFalse(self.calc.es_primo(0))
        self.assertFalse(self.calc.es_primo(1))
        self.assertTrue(self.calc.es_primo(2))
        self.assertTrue(self.calc.es_primo(3))
        self.assertFalse(self.calc.es_primo(4))
        self.assertTrue(self.calc.es_primo(5))
        self.assertFalse(self.calc.es_primo(9))
        self.assertTrue(self.calc.es_primo(13))

if __name__ == '__main__':
    unittest.main()

Ejercicio 2: Pruebas para una clase Usuario

Crea una clase Usuario con los siguientes atributos y métodos, y escribe pruebas unitarias para ella:

  • Atributos: nombre, email, contrasena, activo (booleano)
  • Métodos:
    • cambiar_contrasena(actual, nueva): Cambia la contraseña si la actual es correcta
    • activar_cuenta(): Activa la cuenta
    • desactivar_cuenta(): Desactiva la cuenta
    • es_contraseña_segura(): Devuelve True si la contraseña tiene al menos 8 caracteres, incluyendo mayúsculas, minúsculas y números

Para probar el método es_contraseña_segura, considera usar expresiones regulares o cualquier otro método para verificar los requisitos de seguridad. Asegúrate de probar diferentes casos de contraseñas (demasiado cortas, sin mayúsculas, sin números, etc.).

Python 
import re
import unittest

class Usuario:
    def __init__(self, nombre, email, contrasena, activo=True):
        self.nombre = nombre
        self.email = email
        self.contrasena = contrasena
        self.activo = activo
    
    def cambiar_contrasena(self, actual, nueva):
        if self.contrasena != actual:
            raise ValueError("La contraseña actual no es correcta")
        if not self.es_contraseña_segura(nueva):
            raise ValueError("La nueva contraseña no cumple con los requisitos de seguridad")
        self.contrasena = nueva
        return True
    
    def activar_cuenta(self):
        self.activo = True
        return True
    
    def desactivar_cuenta(self):
        self.activo = False
        return True
    
    def es_contraseña_segura(self, contrasena=None):
        if contrasena is None:
            contrasena = self.contrasena
        
        if len(contrasena) < 8:
            return False
        if not re.search(r'[A-Z]', contrasena):
            return False
        if not re.search(r'[a-z]', contrasena):
            return False
        if not re.search(r'\d', contrasena):
            return False
        return True

class TestUsuario(unittest.TestCase):
    def setUp(self):
        self.usuario = Usuario(
            nombre="Juan Pérez",
            email="juan@ejemplo.com",
            contrasena="Segura123"
        )
    
    def test_inicializacion(self):
        self.assertEqual(self.usuario.nombre, "Juan Pérez")
        self.assertEqual(self.usuario.email, "juan@ejemplo.com")
        self.assertTrue(self.usuario.activo)
    
    def test_cambiar_contrasena(self):
        # Cambio exitoso
        self.assertTrue(self.usuario.cambiar_contrasena("Segura123", "NuevaPass123"))
        self.assertEqual(self.usuario.contrasena, "NuevaPass123")
        
        # Contraseña actual incorrecta
        with self.assertRaises(ValueError):
            self.usuario.cambiar_contrasena("incorrecta", "OtraPass123")
        
        # Nueva contraseña insegura
        with self.assertRaises(ValueError):
            self.usuario.cambiar_contrasena("NuevaPass123", "corta")
    
    def test_activar_desactivar_cuenta(self):
        # Desactivar cuenta
        self.assertTrue(self.usuario.desactivar_cuenta())
        self.assertFalse(self.usuario.activo)
        
        # Reactivar cuenta
        self.assertTrue(self.usuario.activar_cuenta())
        self.assertTrue(self.usuario.activo)
    
    def test_es_contraseña_segura(self):
        # Contraseñas seguras
        self.assertTrue(self.usuario.es_contraseña_segura("Segura123"))
        self.assertTrue(self.usuario.es_contraseña_segura("OtraSegura456"))
        
        # Contraseñas inseguras
        self.assertFalse(self.usuario.es_contraseña_segura("corta"))  # Demasiado corta
        self.assertFalse(self.usuario.es_contraseña_segura("solominusculas"))  # Sin mayúsculas ni números
        self.assertFalse(self.usuario.es_contrase_segura("SOLOMAYUSCULAS"))  # Sin minúsculas ni números
        self.assertFalse(self.usuario.es_contraseña_segura("12345678"))  # Sin letras
        self.assertFalse(self.usuario.es_contraseña_segura("SoloLetras"))  # Sin números

if __name__ == '__main__':
    unittest.main()

Ejercicio 3: Pruebas para una API REST simulada

Escribe pruebas unitarias para la siguiente clase que simula un cliente para una API REST. Usa mocks para simular las respuestas HTTP.

Python 
# api_client.py
import requests

class APIClient:
    def __init__(self, base_url, api_key=None):
        self.base_url = base_url.rstrip('/')
        self.api_key = api_key
        self.session = requests.Session()
        if self.api_key:
            self.session.headers.update({"Authorization": f"Bearer {self.api_key}"})
    
    def get_resource(self, resource_id):
        """Obtiene un recurso por su ID."""
        url = f"{self.base_url}/resources/{resource_id}"
        response = self.session.get(url)
        response.raise_for_status()
        return response.json()
    
    def create_resource(self, data):
        """Crea un nuevo recurso."""
        if not self.api_key:
            raise ValueError("Se requiere una API key para crear recursos")
        url = f"{self.base_url}/resources"
        response = self.session.post(url, json=data)
        response.raise_for_status()
        return response.json()
    
    def update_resource(self, resource_id, data):
        """Actualiza un recurso existente."""
        if not self.api_key:
            raise ValueError("Se requiere una API key para actualizar recursos")
        url = f"{self.base_url}/resources/{resource_id}"
        response = self.session.put(url, json=data)
        response.raise_for_status()
        return response.json()
    
    def delete_resource(self, resource_id):
        """Elimina un recurso."""
        if not self.api_key:
            raise ValueError("Se requiere una API key para eliminar recursos")
        url = f"{self.base_url}/resources/{resource_id}"
        response = self.session.delete(url)
        response.raise_for_status()
        return response.status_code == 204

Usa el decorador @patch para simular las llamadas a requests.Session. Configura los mocks para que devuelvan respuestas simuladas y verifica que los métodos de la clase APIClient se comporten correctamente en diferentes escenarios (éxito, error de autenticación, recurso no encontrado, etc.).

Python 
import unittest
from unittest.mock import patch, MagicMock
import requests
from api_client import APIClient

class TestAPIClient(unittest.TestCase):
    def setUp(self):
        self.base_url = "https://api.ejemplo.com"
        self.api_key = "test-api-key"
        self.client = APIClient(self.base_url, self.api_key)
        self.mock_response = MagicMock()
        self.mock_response.json.return_value = {"id": 1, "name": "Test Resource"}
    
    @patch('requests.Session')
    def test_get_resource_success(self, mock_session_class):
        # Configurar el mock
        mock_session = mock_session_class.return_value
        mock_session.get.return_value = self.mock_response
        
        # Llamar al método bajo prueba
        result = self.client.get_resource(1)
        
        # Verificar que se hizo la petición correcta
        mock_session.get.assert_called_once_with(
            f"{self.base_url}/resources/1"
        )
        
        # Verificar el resultado
        self.assertEqual(result, {"id": 1, "name": "Test Resource"})
    
    @patch('requests.Session')
    def test_get_resource_not_found(self, mock_session_class):
        # Configurar el mock para simular un error 404
        mock_session = mock_session_class.return_value
        mock_response = MagicMock()
        mock_response.raise_for_status.side_effect = requests.HTTPError("404 Not Found")
        mock_session.get.return_value = mock_response
        
        # Verificar que se lanza la excepción
        with self.assertRaises(requests.HTTPError):
            self.client.get_resource(999)
    
    @patch('requests.Session')
    def test_create_resource_success(self, mock_session_class):
        # Configurar el mock
        mock_session = mock_session_class.return_value
        mock_session.post.return_value = self.mock_response
        
        # Datos de prueba
        test_data = {"name": "Nuevo Recurso"}
        
        # Llamar al método bajo prueba
        result = self.client.create_resource(test_data)
        
        # Verificar que se hizo la petición correcta
        mock_session.post.assert_called_once_with(
            f"{self.base_url}/resources",
            json=test_data
        )
        
        # Verificar que se incluyó el encabezado de autorización
        self.assertIn("Authorization", mock_session.headers)
        self.assertEqual(mock_session.headers["Authorization"], f"Bearer {self.api_key}")
        
        # Verificar el resultado
        self.assertEqual(result, {"id": 1, "name": "Test Resource"})
    
    def test_create_resource_missing_api_key(self):
        # Crear un cliente sin API key
        client = APIClient(self.base_url)
        
        # Verificar que se lanza la excepción
        with self.assertRaises(ValueError):
            client.create_resource({"name": "Recurso"})
    
    @patch('requests.Session')
    def test_update_resource_success(self, mock_session_class):
        # Configurar el mock
        mock_session = mock_session_class.return_value
        mock_session.put.return_value = self.mock_response
        
        # Datos de prueba
        test_data = {"name": "Recurso Actualizado"}
        
        # Llamar al método bajo prueba
        result = self.client.update_resource(1, test_data)
        
        # Verificar que se hizo la petición correcta
        mock_session.put.assert_called_once_with(
            f"{self.base_url}/resources/1",
            json=test_data
        )
        
        # Verificar el resultado
        self.assertEqual(result, {"id": 1, "name": "Test Resource"})
    
    @patch('requests.Session')
    def test_delete_resource_success(self, mock_session_class):
        # Configurar el mock
        mock_session = mock_session_class.return_value
        mock_response = MagicMock()
        mock_response.status_code = 204
        mock_session.delete.return_value = mock_response
        
        # Llamar al método bajo prueba
        result = self.client.delete_resource(1)
        
        # Verificar que se hizo la petición correcta
        mock_session.delete.assert_called_once_with(
            f"{self.base_url}/resources/1"
        )
        
        # Verificar el resultado
        self.assertTrue(result)
    
    @patch('requests.Session')
    def test_delete_resource_not_found(self, mock_session_class):
        # Configurar el mock para simular un error 404
        mock_session = mock_session_class.return_value
        mock_session.delete.return_value.raise_for_status.side_effect = requests.HTTPError("404 Not Found")
        
        # Verificar que se lanza la excepción
        with self.assertRaises(requests.HTTPError):
            self.client.delete_resource(999)

if __name__ == '__main__':
    unittest.main()

1.15: Proyecto Integrador Python

Módulo 1: Introducción a Python

Duración: 3-5 horas Fecha: [Fecha]

Objetivos del Proyecto

En esta sesión final del módulo de Python, aplicarás todos los conceptos aprendidos a lo largo del curso para desarrollar un proyecto completo. Este proyecto te permitirá:

  • Integrar múltiples conceptos de programación en Python
  • Desarrollar una aplicación funcional desde cero
  • Practicar la resolución de problemas del mundo real
  • Mejorar tus habilidades de depuración y prueba de código
  • Prepararte para proyectos más avanzados en el campo de la inteligencia artificial

Instrucciones Generales

Selecciona uno de los proyectos propuestos a continuación. Cada proyecto tiene diferentes niveles de dificultad y cubre diferentes aspectos de la programación en Python. Sigue estos pasos:

  1. Selecciona un proyecto que se ajuste a tus intereses y nivel de habilidad.
  2. Planifica tu solución antes de comenzar a codificar.
  3. Divide el problema en partes más pequeñas y manejables.
  4. Escribe código limpio y bien documentado.
  5. Prueba cada función o módulo a medida que avanzas.
  6. Refactoriza tu código para mejorar su calidad.
  7. Prepara una presentación de tu proyecto para compartir con la clase.

Selecciona un proyecto:

Sistema de Gestión de Tareas

Nivel: Principiante
Listas Archivos Funciones CRUD

Descripción:

Desarrolla una aplicación de línea de comandos para gestionar tareas pendientes. Los usuarios podrán agregar, ver, marcar como completadas y eliminar tareas. Las tareas se guardarán en un archivo para persistencia de datos.

Características principales:

  • Agregar nuevas tareas con descripción y fecha límite
  • Listar todas las tareas con su estado (pendiente/completada)
  • Marcar tareas como completadas
  • Eliminar tareas
  • Guardar y cargar tareas desde un archivo
  • Filtrar tareas por estado o fecha

Desafíos adicionales (opcional):

  • Agregar categorías a las tareas
  • Implementar búsqueda de tareas por palabra clave
  • Agregar prioridad a las tareas
  • Crear una interfaz gráfica con Tkinter o PyQt

Recursos útiles:

  • Manejo de archivos en Python
  • Manejo de fechas con el módulo datetime
  • Manejo de listas y diccionarios

Estructura sugerida del proyecto:

gestor_tareas/ ├── main.py # Punto de entrada de la aplicación ├── tareas.py # Lógica de gestión de tareas ├── archivo.py # Funciones para guardar/cargar desde archivo └── README.md # Documentación del proyecto

Juego del Ahorcado

Nivel: Principiante/Intermedio
Cadenas Bucles Funciones Juegos

Descripción:

Implementa el clásico juego del ahorcado donde el jugador debe adivinar una palabra letra por letra antes de que se complete el dibujo del ahorcado.

Características principales:

  • Selección aleatoria de palabras de una lista o archivo
  • Interfaz de texto que muestra el estado actual del juego
  • Seguimiento de letras adivinadas y disponibles
  • Contador de intentos restantes
  • Detección de victoria/derrota
  • Opción para jugar nuevamente

Desafíos adicionales (opcional):

  • Agregar categorías de palabras (animales, países, etc.)
  • Implementar un sistema de puntuación
  • Guardar puntuaciones altas en un archivo
  • Agregar una interfaz gráfica con Tkinter o Pygame
  • Incluir pistas o definiciones de las palabras

Recursos útiles:

  • Manejo de cadenas en Python
  • Bucles y condicionales
  • Módulo random para selección aleatoria

Estructura sugerida del proyecto:

ahorcado/ ├── main.py # Punto de entrada del juego ├── juego.py # Lógica principal del juego ├── palabras.py # Gestión de palabras y categorías ├── interfaz.py # Funciones de interfaz de usuario └── README.md # Documentación del proyecto

Gestor Financiero Personal

Nivel: Intermedio
POO Archivos CSV/JSON Módulos Análisis de datos

Descripción:

Desarrolla una aplicación para gestionar finanzas personales que permita a los usuarios registrar ingresos y gastos, categorizar transacciones y generar informes.

Características principales:

  • Registrar ingresos y gastos con descripción, monto, fecha y categoría
  • Ver resumen financiero (balance, ingresos totales, gastos totales)
  • Filtrar transacciones por categoría, fecha o tipo (ingreso/gasto)
  • Exportar datos a CSV o JSON
  • Cargar datos desde archivo
  • Generar gráficos simples de gastos por categoría

Desafíos adicionales (opcional):

  • Implementar un sistema de presupuestos por categoría
  • Agregar gráficos más avanzados con Matplotlib
  • Implementar autenticación de usuarios
  • Crear una API REST para acceder a los datos
  • Desarrollar una interfaz web con Flask o Django

Recursos útiles:

  • Programación orientada a objetos en Python
  • Manejo de archivos CSV/JSON
  • Módulo datetime para manejo de fechas
  • Librerías como pandas para análisis de datos

Estructura sugerida del proyecto:

gestor_financiero/ ├── main.py # Punto de entrada de la aplicación ├── transaccion.py # Clase Transacción ├── gestor.py # Lógica principal del gestor ├── almacenamiento.py # Manejo de archivos ├── informes.py # Generación de informes └── README.md # Documentación del proyecto

Sistema de Gestión de Contenidos (CMS) para Blog

Nivel: Avanzado
Flask/Django Base de datos HTML/CSS CRUD

Descripción:

Desarrolla un sistema de gestión de contenidos simple para un blog que permita crear, leer, actualizar y eliminar publicaciones.

Características principales:

  • Interfaz web para gestionar publicaciones
  • Almacenamiento de publicaciones en una base de datos
  • Soporte para formato Markdown en el contenido
  • Sistema de categorías y etiquetas
  • Búsqueda de publicaciones
  • Paginación de resultados

Desafíos adicionales (opcional):

  • Implementar un sistema de comentarios
  • Agregar autenticación de usuarios
  • Implementar un sistema de roles (admin, editor, lector)
  • Agregar carga de imágenes
  • Crear una API REST para el frontend

Recursos útiles:

  • Framework web (Flask o Django)
  • Bases de datos SQL (SQLite, PostgreSQL)
  • ORM (SQLAlchemy, Django ORM)
  • HTML/CSS/JavaScript básico
  • Manejo de formularios

Estructura sugerida del proyecto (Flask):

blog_cms/ ├── app/ │ ├── __init__.py │ ├── models.py # Modelos de datos │ ├── routes.py # Rutas de la aplicación │ ├── forms.py # Formularios │ ├── static/ # Archivos estáticos (CSS, JS, imágenes) │ └── templates/ # Plantillas HTML ├── config.py # Configuración ├── requirements.txt # Dependencias └── run.py # Punto de entrada

Analizador de Archivos

Nivel: Intermedio/Avanzado
Sistema de archivos Expresiones regulares Interfaz de línea de comandos Procesamiento de texto

Descripción:

Desarrolla una herramienta de línea de comandos que analice archivos y directorios, proporcionando información detallada y realizando operaciones de búsqueda y procesamiento.

Características principales:

  • Explorar directorios y mostrar información de archivos
  • Buscar archivos por nombre, extensión o contenido
  • Obtener estadísticas de archivos (tamaño, líneas, palabras, caracteres)
  • Buscar y reemplazar texto en archivos
  • Generar informes en diferentes formatos (texto, CSV, JSON)

Desafíos adicionales (opcional):

  • Agregar soporte para expresiones regulares en búsquedas
  • Implementar un modo interactivo
  • Agregar opciones de filtrado avanzado
  • Crear una interfaz gráfica con Tkinter o PyQt
  • Agregar soporte para archivos comprimidos

Recursos útiles:

  • Módulo os y pathlib para manejo de archivos
  • Módulo argparse para argumentos de línea de comandos
  • Expresiones regulares con el módulo re
  • Manejo de diferentes codificaciones de archivos

Estructura sugerida del proyecto:

analizador_archivos/ ├── main.py # Punto de entrada ├── scanner.py # Exploración de archivos ├── analyzer.py # Análisis de contenido ├── search.py # Búsqueda y reemplazo ├── report.py # Generación de informes └── README.md # Documentación

Criterios de Evaluación

Tu proyecto será evaluado según los siguientes criterios:

Criterio Ponderación Descripción
Funcionalidad 30% El proyecto cumple con todos los requisitos funcionales especificados.
Calidad del código 25% El código está bien estructurado, es legible y sigue las mejores prácticas de Python.
Documentación 15% El código está adecuadamente documentado con comentarios y docstrings.
Manejo de errores 15% El código maneja adecuadamente los errores y casos extremos.
Presentación 15% La presentación del proyecto es clara y profesional.

Entrega del Proyecto

Para entregar tu proyecto, sigue estos pasos:

  1. Crea un repositorio en GitHub para tu proyecto.
  2. Agrega todos los archivos necesarios al repositorio.
  3. Crea un archivo README.md con la documentación del proyecto.
  4. Prepara una presentación de 5-10 minutos mostrando tu proyecto.
  5. Comparte el enlace al repositorio y la presentación con el instructor.
Consejo: Asegúrate de probar exhaustivamente tu proyecto antes de la entrega. Pídele a un compañero que lo pruebe también para identificar posibles problemas.
